<task type="auto">
  <name>Task 1: Define the complete Prisma schema for all 10 phases</name>
  <files>
    apps/api/prisma/schema.prisma
  </files>
  <action>
    Install Prisma: `pnpm --filter api add @prisma/client` and `pnpm --filter api add -D prisma`.
    Initialize Prisma: `pnpm --filter api exec prisma init --datasource-provider mysql`.

    Create the complete schema in `apps/api/prisma/schema.prisma` with MySQL datasource and prisma-client-js generator. Use `env("DATABASE_URL")` for the connection string.

    **Naming conventions:** All models use PascalCase, all fields use camelCase, all @@map to snake_case table names, all @map to snake_case column names. Use `@id @default(cuid())` for all primary keys. Use `@default(now())` for createdAt and `@updatedAt` for updatedAt on every model.

    **Models to define (grouped by domain):**

    **Auth (Phase 1):**
    - `User`: id, email (unique), name, passwordHash, role (enum: USER/SUBSCRIBER/ADMIN), avatarUrl?, bio?, acceptedTermsAt, createdAt, updatedAt + all relations
    - `RefreshToken`: id, userId, tokenHash, tokenFamily, revoked (default false), expiresAt, createdAt + index on userId, tokenHash
    - `PasswordReset`: id, userId, tokenHash (unique), expiresAt, usedAt?, createdAt + index on userId

    **Catalog (Phase 2):**
    - `CatalogEntry`: id, title, author?, publisher?, imprint?, barcode?, isbn?, description? (Text), coverImageUrl?, approvalStatus (enum: DRAFT/PENDING/APPROVED/REJECTED), rejectionReason?, averageRating (Decimal 3,2 default 0), ratingCount (Int default 0), createdById, seriesId?, volumeNumber?, editionNumber?, createdAt, updatedAt
    - `Series`: id, title, description? (Text), totalEditions (Int), createdAt, updatedAt
    - `Category`: id, name (unique), slug (unique), description?, createdAt, updatedAt
    - `Tag`: id, name (unique), slug (unique), createdAt, updatedAt
    - `Character`: id, name, slug (unique), description?, imageUrl?, createdAt, updatedAt
    - Junction tables: `CatalogCategory`, `CatalogTag`, `CatalogCharacter` (many-to-many)

    **Collection (Phase 3):**
    - `CollectionItem`: id, userId, catalogEntryId, quantity (Int default 1), pricePaid? (Decimal 10,2), condition (enum: NEW/VERY_GOOD/GOOD/FAIR/POOR), notes? (Text), isRead (Boolean default false), readAt?, isForSale (Boolean default false), salePrice? (Decimal 10,2), createdAt, updatedAt

    **Cart & Marketplace (Phase 4):**
    - `CartItem`: id, userId, collectionItemId, reservedAt, expiresAt, createdAt
    - `ShippingAddress`: id, userId, label?, street, number, complement?, neighborhood, city, state, zipCode, isDefault (Boolean default false), createdAt, updatedAt
    - `ShippingMethod`: id, name, description?, isActive (Boolean default true), createdAt, updatedAt

    **Orders (Phase 4):**
    - `Order`: id, orderNumber (unique), buyerId, status (enum: PENDING/PAID/PROCESSING/SHIPPED/DELIVERED/COMPLETED/CANCELLED/DISPUTED), shippingAddressSnapshot (Json), totalAmount (Decimal 10,2), createdAt, updatedAt
    - `OrderItem`: id, orderId, collectionItemId, sellerId, priceSnapshot (Decimal 10,2), commissionRateSnapshot (Decimal 5,4), commissionAmountSnapshot (Decimal 10,2), sellerNetSnapshot (Decimal 10,2), status (enum matching order item flow), trackingCode?, carrier?, shippedAt?, deliveredAt?, cancelledAt?, createdAt, updatedAt

    **Payments (Phase 5):**
    - `Payment`: id, orderId, method (enum: PIX), providerPaymentId?, providerStatus?, amount (Decimal 10,2), pixQrCode? (Text), pixCopyPaste? (Text), pixExpiresAt?, paidAt?, refundedAmount? (Decimal 10,2), createdAt, updatedAt
    - `WebhookEvent`: id, provider (String), eventId (String), eventType (String), payload (Json), processedAt?, createdAt + unique constraint on [provider, eventId]

    **Commissions (Phase 4-5):**
    - `CommissionConfig`: id, planType (enum matching UserRole or a PlanType enum), rate (Decimal 5,4), minRate? (Decimal 5,4), maxRate? (Decimal 5,4), isActive (Boolean default true), createdAt, updatedAt

    **Seller Banking (Phase 5):**
    - `BankAccount`: id, userId, bankName, branchNumber, accountNumber, cpf, holderName, accountType (enum: CHECKING/SAVINGS), isPrimary (Boolean default false), createdAt, updatedAt

    **Subscriptions (Phase 6):**
    - `Subscription`: id, userId, stripeCustomerId?, stripeSubscriptionId?, planType (enum: FREE/BASIC), status (enum: ACTIVE/CANCELLED/PAST_DUE/TRIALING), currentPeriodStart?, currentPeriodEnd?, cancelledAt?, createdAt, updatedAt
    - `PlanConfig`: id, planType, name, price (Decimal 10,2), billingInterval (enum: MONTHLY/QUARTERLY/SEMIANNUAL/ANNUAL), collectionLimit (Int), commissionRate (Decimal 5,4), trialDays (Int default 0), isActive (Boolean default true), createdAt, updatedAt

    **Community (Phase 7):**
    - `Review`: id, userId, catalogEntryId?, sellerId?, rating (Int), text? (Text), createdAt, updatedAt + unique on [userId, catalogEntryId] and [userId, sellerId, orderId]
    - `Comment`: id, userId, catalogEntryId, parentId? (self-referential for one level nesting), content (Text), likesCount (Int default 0), createdAt, updatedAt
    - `CommentLike`: id, userId, commentId, createdAt + unique on [userId, commentId]
    - `Favorite`: id, userId, catalogEntryId, createdAt + unique on [userId, catalogEntryId]

    **Notifications (Phase 7):**
    - `Notification`: id, userId, type (enum: WELCOME/PAYMENT_CONFIRMED/ORDER_SHIPPED/ITEM_SOLD/PASSWORD_RESET/DISPUTE_OPENED/DISPUTE_RESPONDED/DISPUTE_RESOLVED/SUBSCRIPTION_PAYMENT_FAILED/SUBSCRIPTION_EXPIRED), title, message (Text), isRead (Boolean default false), readAt?, metadata? (Json), createdAt
    - `NotificationPreference`: id, userId, type (same enum), enabled (Boolean default true), createdAt, updatedAt + unique on [userId, type]

    **Deals (Phase 9):**
    - `PartnerStore`: id, name, slug (unique), affiliateTag, baseUrl, logoUrl?, isActive (Boolean default true), createdAt, updatedAt
    - `Deal`: id, storeId, type (enum: COUPON/PROMOTION), title, description? (Text), couponCode?, discount?, bannerUrl?, affiliateBaseUrl, categoryId?, startsAt?, expiresAt?, isActive (Boolean default true), createdAt, updatedAt
    - `ClickLog`: id, dealId, userId?, ipAddress, userAgent? (Text), createdAt + index on [dealId, userId, createdAt] for deduplication

    **Homepage (Phase 9):**
    - `HomepageSection`: id, type (enum: BANNER_CAROUSEL/CATALOG_HIGHLIGHTS/DEALS_OF_DAY/FEATURED_COUPONS), title?, sortOrder (Int), isVisible (Boolean default true), contentRefs? (Json), createdAt, updatedAt

    **Disputes (Phase 8):**
    - `Dispute`: id, orderId, orderItemId, buyerId, sellerId, reason (enum: NOT_RECEIVED/DIFFERENT_FROM_LISTING/DAMAGED_IN_TRANSIT/NOT_SHIPPED_ON_TIME), description (Text), status (enum: OPEN/IN_MEDIATION/RESOLVED_REFUND/RESOLVED_PARTIAL_REFUND/RESOLVED_NO_REFUND/CANCELLED), resolution? (Text), resolvedById?, resolvedAt?, refundAmount? (Decimal 10,2), createdAt, updatedAt
    - `DisputeEvidence`: id, disputeId, submittedById, imageUrl, description?, createdAt
    - `DisputeMessage`: id, disputeId, senderId, message (Text), createdAt

    **Legal (Phase 10):**
    - `LegalDocument`: id, type (enum: TERMS_OF_USE/PRIVACY_POLICY/SELLER_TERMS/PAYMENT_POLICY/RETURNS_POLICY/SHIPPING_POLICY/CANCELLATION_POLICY/COOKIES_POLICY), version (Int), content (LongText/MediumText), dateOfEffect, isMandatory (Boolean default false), createdAt
    - `LegalAcceptance`: id, userId, documentId, acceptedAt, ipAddress, createdAt

    **LGPD (Phase 10):**
    - `DataRequest`: id, userId, type (enum: ACCESS/CORRECTION/DELETION/EXPORT), status (enum: PENDING/PROCESSING/COMPLETED/REJECTED), details? (Text), completedAt?, createdAt, updatedAt

    **Contact (Phase 10):**
    - `ContactMessage`: id, name, email, category (enum: SUGGESTION/PROBLEM/PARTNERSHIP/OTHER), subject, message (Text), isRead (Boolean default false), isResolved (Boolean default false), readAt?, resolvedAt?, createdAt

    **Key constraints per research:**
    - All monetary values: `@db.Decimal(10, 2)`
    - All commission rates: `@db.Decimal(5, 4)`
    - All ratings: `@db.Decimal(3, 2)` for averages, `Int` for individual 1-5 ratings
    - All text content that may be long: `@db.Text` or `@db.MediumText`
    - Use `@@map("snake_case_table_name")` for every model
    - Use `@map("snake_case_column_name")` for multi-word field names
    - Foreign keys nullable where the relation is optional (e.g., Review.sellerId)
    - Cascade deletes on User relations where appropriate (RefreshToken, PasswordReset)
    - Appropriate `@@index` on frequently queried foreign keys
    - `@@unique` constraints for business rules (one review per user per catalog, etc.)

<task type="auto">
  <name>Task 2: Generate initial migration and create seed script</name>
  <files>
    apps/api/prisma/seed.ts
    apps/api/package.json
  </files>
  <action>
    1. Create a `.env` file in `apps/api/` with `DATABASE_URL="mysql://root:password@localhost:3306/comicstrunk"` (local dev only — production uses cPanel env vars)

    2. Generate the initial migration:
       ```
       pnpm --filter api exec prisma migrate dev --name init
       ```
       This creates the `prisma/migrations/` directory with the initial SQL.
       If the local MySQL is not running, use `prisma migrate dev --create-only` to generate migration SQL without applying it. Verify the SQL is correct.

    3. Create `apps/api/prisma/seed.ts`:
       - Import PrismaClient and bcryptjs
       - Create a default admin user:
         - email: "admin@comicstrunk.com", name: "Admin", role: ADMIN
         - password: hash of "Admin123!" (development only)
         - acceptedTermsAt: new Date()
       - Create default plan configs:
         - FREE plan: collectionLimit 50, commissionRate 0.10, price 0, billingInterval MONTHLY
         - BASIC plan: collectionLimit 200, commissionRate 0.08, price (placeholder e.g., 14.90), billingInterval MONTHLY
       - Use upsert to make seed idempotent

    4. Add prisma seed config to `apps/api/package.json`:
       ```json
       "prisma": {
         "seed": "tsx prisma/seed.ts"
       }
       ```

    5. Add convenience scripts to `apps/api/package.json`:
       - "db:migrate": "prisma migrate dev"
       - "db:migrate:deploy": "prisma migrate deploy"
       - "db:seed": "prisma db seed"
       - "db:studio": "prisma studio"
       - "db:generate": "prisma generate"

    6. Run `pnpm --filter api exec prisma generate` to generate the Prisma Client
  </action>
