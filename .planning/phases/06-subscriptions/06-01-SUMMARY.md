# Phase 06 Plan 01: Subscription API & Stripe SDK Summary

Stripe SDK abstraction with conditional initialization, 5 subscription API endpoints (plans, status, checkout, portal, cancel), PlanConfig migration for stripePriceId, and seed data for FREE + 4 BASIC billing intervals.

## What Was Built

### Stripe SDK Abstraction (`apps/api/src/shared/lib/stripe.ts`)
- Conditional Stripe client initialization based on `STRIPE_SECRET_KEY` env var
- `isStripeConfigured()` boolean guard for feature toggling
- `verifyWebhookSignature()` with dev-mode fallback (skips verification when `STRIPE_WEBHOOK_SECRET` not set)
- API version: `2026-02-25.clover` (matches stripe@20.4.0)

### Contract Schemas (`packages/contracts/src/subscription.ts`)
- `createCheckoutSchema` for POST /checkout validation
- `planConfigResponseSchema` and `subscriptionStatusResponseSchema` for response typing
- Exported enums: `PlanType`, `BillingInterval`, `SubscriptionStatus`
- Inferred types: `CreateCheckoutInput`, `PlanConfigResponse`, `SubscriptionStatusResponse`

### Subscription Service (`apps/api/src/modules/subscriptions/subscriptions.service.ts`)
- `listActivePlans()` — returns all active PlanConfig rows ordered by type and price
- `getSubscriptionStatus(userId)` — returns current subscription or synthetic FREE status with plan limits
- `createCheckoutSession(userId, planConfigId)` — creates Stripe Checkout Session with customer management
- `createPortalSession(userId)` — creates Stripe Customer Portal session
- `cancelSubscription(userId)` — schedules end-of-period cancellation via `cancel_at_period_end`

### Subscription Routes (`apps/api/src/modules/subscriptions/subscriptions.routes.ts`)
- `GET /plans` (public) — list active plan configs
- `GET /status` (auth) — current user subscription status
- `POST /checkout` (auth) — create checkout session
- `POST /portal` (auth) — create customer portal session
- `POST /cancel` (auth) — cancel subscription

### Webhook Infrastructure (auto-generated by linter, validates plan 06-02)
- `stripe-webhook.routes.ts` — POST handler with raw body and Stripe signature verification
- `stripe-webhook.service.ts` — full event processing with idempotency guard (WebhookEvent pattern)
  - Handles: checkout.session.completed, subscription.updated, subscription.deleted, invoice.payment_failed, invoice.paid
  - Upgrades user role to SUBSCRIBER on checkout completion
  - Downgrades to FREE on subscription deletion with notification

### Database Migration
- Added `stripe_price_id VARCHAR(191) NULL` column to `plan_configs` table
- Fixed pre-existing duplicate migration (20260223215544) by converting to no-op

### Seed Data
- 5 PlanConfig rows seeded with deterministic IDs:
  - `plan-free-monthly`: FREE, R$0.00, 50 items, 10% commission, 0 trial days
  - `plan-basic-monthly`: BASIC, R$9.90, 200 items, 8% commission, 7 trial days
  - `plan-basic-quarterly`: BASIC, R$24.90, 200 items, 8% commission, 7 trial days
  - `plan-basic-semiannual`: BASIC, R$44.90, 200 items, 8% commission, 7 trial days
  - `plan-basic-annual`: BASIC, R$79.90, 200 items, 8% commission, 7 trial days
- Upsert now updates existing records (name, price, limits, isActive) for true idempotency
