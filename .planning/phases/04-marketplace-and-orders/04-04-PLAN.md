  <name>Task 1: Create order state machine and order service</name>
  <files>

  </files>
  <action>
    **apps/api/src/shared/lib/order-state-machine.ts:**
    - Define `ORDER_TRANSITIONS` Record mapping each OrderStatus to allowed next statuses:
      PENDING -> [PAID, CANCELLED]
      PAID -> [PROCESSING, CANCELLED]
      PROCESSING -> [SHIPPED, CANCELLED]
      SHIPPED -> [DELIVERED, DISPUTED]
      DELIVERED -> [COMPLETED, DISPUTED]
      COMPLETED -> []
      CANCELLED -> []
      DISPUTED -> [COMPLETED, CANCELLED]
    - Define `ORDER_ITEM_TRANSITIONS` Record:
      PENDING -> [PAID, CANCELLED]
      PAID -> [PROCESSING, CANCELLED]
      PROCESSING -> [SHIPPED, CANCELLED]
      SHIPPED -> [DELIVERED, DISPUTED]
      DELIVERED -> [COMPLETED, DISPUTED]
      COMPLETED -> []
      CANCELLED -> []
      DISPUTED -> [COMPLETED, CANCELLED, REFUNDED]
      REFUNDED -> []
    - Export `assertOrderTransition(current, next)` and `assertOrderItemTransition(current, next)` that throw BadRequestError if transition is invalid

    **apps/api/src/modules/orders/orders.service.ts:**

    `createOrder(buyerId: string, shippingAddressId: string)`:
    Use `prisma.$transaction` (interactive):
    1. Fetch active cart items (expiresAt > now) for buyer, include collectionItem + collectionItem.user
    2. Throw BadRequestError('Cart is empty') if no items
    3. Fetch and validate shipping address (exists, belongs to buyer)
    4. Generate order number using generateOrderNumber() — wrap in retry loop (max 3) catching unique constraint violations
    5. For each cart item:
       a. Get seller's current plan (query Subscription where userId = item.collectionItem.userId, status ACTIVE; default to FREE)
       b. Call getCommissionRate(planType) from commission service
       c. Call calculateCommission(salePrice, rate) using roundCurrency
       d. Build OrderItem data: collectionItemId, sellerId, priceSnapshot, commissionRateSnapshot, commissionAmountSnapshot, sellerNetSnapshot
    6. Calculate totalAmount as sum of all priceSnapshots
    7. Create Order with orderNumber, buyerId, totalAmount, shippingAddressSnapshot (JSON.parse(JSON.stringify(address)) to snapshot all fields), and nested OrderItem creates
    8. Delete all buyer's cart items (clear cart)
    9. Return order with items

    `getOrder(userId: string, orderId: string)`:
    - Find order by id, include orderItems with collectionItem->catalogEntry (for title, cover)
    - Verify order.buyerId === userId OR userId is a seller on one of the order items
    - Throw ForbiddenError if neither buyer nor seller
    - Return order

    `getOrderByNumber(userId: string, orderNumber: string)`:
    - Same as getOrder but lookup by orderNumber

    `listBuyerOrders(buyerId: string, filters: { status?, page, limit })`:
    - Find orders where buyerId, optional status filter
    - Include orderItems count and basic info
    - Paginate and return with sendPaginated format

    `listSellerOrders(sellerId: string, filters: { status?, page, limit })`:
    - Find orders that have at least one orderItem with sellerId
    - Include only the seller's order items (not other sellers' items)
    - Paginate

    `updateOrderItemStatus(userId: string, orderItemId: string, newStatus: string)`:
    - Find order item, include order
    - Verify caller is buyer (for DELIVERED->COMPLETED) or seller (for PAID->PROCESSING) or admin
    - Call assertOrderItemTransition(currentStatus, newStatus)
    - Update status + relevant timestamp (shippedAt, deliveredAt, cancelledAt)
    - If all items in the order reach COMPLETED, auto-update order status to COMPLETED
    - If all items CANCELLED, auto-update order to CANCELLED
    - When item reaches COMPLETED: set collectionItem.isForSale = false, salePrice = null

    `cancelOrder(userId: string, orderId: string)`:
    - Find order, verify buyer owns it
    - assertOrderTransition(current, 'CANCELLED')
    - Update all order items to CANCELLED (only those not already COMPLETED/SHIPPED/DELIVERED)
    - Update order status to CANCELLED
  </action>

  </done>

    - `PATCH /items/:itemId/status` — Update order item status. Validate body with updateOrderItemStatusSchema. Call updateOrderItemStatus.
