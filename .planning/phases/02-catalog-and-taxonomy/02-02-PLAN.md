  <action>
Follow the exact same module pattern as `auth` — routes.ts wires middleware, service.ts has all business logic with Prisma queries.

**Series module (`apps/api/src/modules/series/`):**

`series.service.ts`:
- `listSeries(filters: SeriesSearchInput)`: Prisma findMany with optional `title` contains filter, pagination (skip/take), count for total. Include `_count: { select: { catalogEntries: true } }` to show edition count. Order by title asc.
- `getSeriesById(id: string)`: findUnique with `include: { catalogEntries: { where: { approvalStatus: 'APPROVED' }, orderBy: { editionNumber: 'asc' }, select: { id, title, volumeNumber, editionNumber, coverImageUrl, author, publisher, averageRating, ratingCount } } }`. Throw NotFoundError if not found.
- `createSeries(data: CreateSeriesInput)`: Prisma create.
- `updateSeries(id: string, data: UpdateSeriesInput)`: findUnique first (throw NotFoundError if missing), then update.
- `deleteSeries(id: string)`: findUnique first (throw NotFoundError if missing). Check if series has catalog entries — if so, throw BadRequestError("Cannot delete series with existing catalog entries"). Otherwise delete.

`series.routes.ts`:
- `GET /` — public, validate query with seriesSearchSchema, call listSeries, return with sendPaginated
- `GET /:id` — public, call getSeriesById, return with sendSuccess
- `POST /` — authenticate, authorize('ADMIN'), validate body with createSeriesSchema, call createSeries, return 201
- `PUT /:id` — authenticate, authorize('ADMIN'), validate body with updateSeriesSchema, call updateSeries, return sendSuccess
- `DELETE /:id` — authenticate, authorize('ADMIN'), call deleteSeries, return sendSuccess with `{ deleted: true }`
- Export as `seriesRoutes: Router`

**Categories module (`apps/api/src/modules/categories/`):**

`categories.service.ts`:
- `listCategories()`: findMany ordered by name asc. Include `_count: { select: { catalogEntries: true } }`.
- `getCategoryById(id: string)`: findUnique, throw NotFoundError if missing.
- `createCategory(data: CreateCategoryInput)`: Generate slug via `uniqueSlug(data.name, 'category')`. Prisma create with name, slug, description.
- `updateCategory(id: string, data: UpdateCategoryInput)`: findUnique first. If name is being changed, regenerate slug via `uniqueSlug(data.name, 'category', id)`. Update.
- `deleteCategory(id: string)`: findUnique first. Check if has catalog entries — if so, throw BadRequestError. Otherwise delete.

`categories.routes.ts`:
- `GET /` — public, listCategories, sendSuccess
- `GET /:id` — public, getCategoryById, sendSuccess
- `POST /` — authenticate, authorize('ADMIN'), validate createCategorySchema, create, 201
- `PUT /:id` — authenticate, authorize('ADMIN'), validate updateCategorySchema, update, sendSuccess
- `DELETE /:id` — authenticate, authorize('ADMIN'), delete, sendSuccess
- Export as `categoriesRoutes: Router`

All routes use try/catch wrapping with `next(err)` following the auth module pattern. All service methods import prisma from `../../shared/lib/prisma`. All route exports use explicit `Router` type annotation per 01-04 decision.
