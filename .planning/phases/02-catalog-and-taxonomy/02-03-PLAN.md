  <name>Task 1: Create catalog service with CRUD, approval state machine, image upload, and junction table management</name>
  <files>

  <action>
Create `apps/api/src/modules/catalog/catalog.service.ts` with the following functions:

**CRUD Operations:**

`createCatalogEntry(data: CreateCatalogEntryInput & { createdById: string })`:
- Extract categoryIds, tagIds, characterIds from data (destructure them out)
- Create with Prisma including junction table associations:
  - `categories: { create: categoryIds?.map(id => ({ categoryId: id })) }` (if categoryIds provided)
  - Same pattern for tags and characters
- Entry starts with `approvalStatus: 'DRAFT'` (Prisma default)
- Include full relations in response (series, categories.category, tags.tag, characters.character)
- Return created entry

`getCatalogEntryById(id: string, publicOnly: boolean = true)`:
- findUnique with full relation includes
- If `publicOnly` is true, verify `approvalStatus === 'APPROVED'` — if not, throw NotFoundError (do NOT expose non-approved entries to public)
- If `publicOnly` is false (admin), return regardless of status
- Throw NotFoundError if not found

`updateCatalogEntry(id: string, data: UpdateCatalogEntryInput)`:
- findUnique first, throw NotFoundError if missing
- Extract categoryIds, tagIds, characterIds
- Build update data for scalar fields
- For junction tables, use transaction:
  - Delete existing: `prisma.catalogCategory.deleteMany({ where: { catalogEntryId: id } })`
  - Create new: `prisma.catalogCategory.createMany({ data: categoryIds.map(cid => ({ catalogEntryId: id, categoryId: cid })) })`
  - Same for tags and characters
  - Wrap junction updates + entry update in `prisma.$transaction`
- Only update junction tables if the corresponding IDs array is provided (not undefined)
- Return updated entry with full includes

`deleteCatalogEntry(id: string)`:
- findUnique first, throw NotFoundError if missing
- Delete (cascade handles junction tables via schema)

**Approval State Machine:**

Define `VALID_TRANSITIONS` map:
```typescript
const VALID_TRANSITIONS: Record<string, string[]> = {
  DRAFT: ['PENDING'],
  PENDING: ['APPROVED', 'REJECTED'],
  APPROVED: ['PENDING'],    // Can send back for re-review
  REJECTED: ['DRAFT'],      // Author can revise and resubmit
};
```

`updateApprovalStatus(id: string, action: 'submit' | 'approve' | 'reject', rejectionReason?: string)`:
- Map actions to target statuses: submit -> PENDING, approve -> APPROVED, reject -> REJECTED
- findUnique the entry, throw NotFoundError if missing
- Check `VALID_TRANSITIONS[entry.approvalStatus]` includes target status, throw BadRequestError if not
- If rejecting, require rejectionReason (throw BadRequestError if missing)
- Update: set approvalStatus, set rejectionReason (null unless rejecting)
- Return updated entry

**Cover Image Upload:**

`uploadCoverImage(id: string, buffer: Buffer)`:
- findUnique the entry, throw NotFoundError if missing
- If entry already has a coverImageUrl and Cloudinary is configured, delete old image via `deleteImage`
- Upload new image via `uploadImage(buffer, 'comicstrunk/covers')` from cloudinary helper
- Update entry with new `coverImageUrl`
- Return updated entry

**Admin List:**

`listCatalogEntries(params: { page: number; limit: number; approvalStatus?: string })`:
- Build where clause: if approvalStatus provided, filter by it; otherwise return all (admin view)
- findMany with pagination, full includes, order by createdAt desc
- Count for total
- Return entries + total

Create a helper `catalogIncludes()` that returns the standard include object:
```typescript
{ series: true, categories: { include: { category: true } }, tags: { include: { tag: true } }, characters: { include: { character: true } }, createdBy: { select: { id: true, name: true, email: true } } }
```
Use this in all queries that return catalog entries.

Create `apps/api/src/modules/catalog/catalog.routes.ts`:

**Public endpoints (no auth):**
- `GET /` — For public use: call getCatalogEntries with `approvalStatus: 'APPROVED'` filter (this is the public browse endpoint; search functionality is in Plan 04)
  - Parse page/limit from query, use paginationSchema from contracts
  - Return with sendPaginated
- `GET /:id` — call getCatalogEntryById with `publicOnly: true`, sendSuccess

**Admin CRUD endpoints:**
- `POST /` — authenticate, authorize('ADMIN'), validate body with createCatalogEntrySchema, call createCatalogEntry with `req.user.id` as createdById, return 201
- `PUT /:id` — authenticate, authorize('ADMIN'), validate body with updateCatalogEntrySchema, call updateCatalogEntry, sendSuccess
- `DELETE /:id` — authenticate, authorize('ADMIN'), call deleteCatalogEntry, sendSuccess

**Admin approval endpoints:**
- `PATCH /:id/submit` — authenticate, authorize('ADMIN'), call updateApprovalStatus(id, 'submit'), sendSuccess
- `PATCH /:id/approve` — authenticate, authorize('ADMIN'), call updateApprovalStatus(id, 'approve'), sendSuccess
- `PATCH /:id/reject` — authenticate, authorize('ADMIN'), validate body for rejectionReason (z.object({ rejectionReason: z.string().min(1).max(2000) })), call updateApprovalStatus(id, 'reject', body.rejectionReason), sendSuccess

**Admin image upload:**
- `POST /:id/cover` — authenticate, authorize('ADMIN'), uploadSingle('cover') middleware, then:
  - If no req.file, throw BadRequestError('No file provided')
  - Call uploadCoverImage(req.params.id, req.file.buffer)
  - sendSuccess with updated entry

**Admin list with status filter:**
- `GET /admin/list` — authenticate, authorize('ADMIN'), parse query (page, limit, approvalStatus optional), call listCatalogEntries, sendPaginated
  - IMPORTANT: This route MUST be defined BEFORE `GET /:id` to prevent Express matching "admin" as an id parameter

Export as `catalogRoutes: Router`

**Register in create-app.ts:**
Add import and `app.use('/api/v1/catalog', catalogRoutes)` after the taxonomy routes (or after auth/users if taxonomy routes are not yet registered — this plan runs in parallel with 02-02).

Verify: `pnpm --filter api type-check` and `pnpm --filter api build` must pass.
