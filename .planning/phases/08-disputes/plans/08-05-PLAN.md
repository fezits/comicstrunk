# Plan 08-05: Admin Dispute UI

## Wave: 3 (parallel with 08-04, depends on 08-01 + 08-02)

## Goal
Create admin pages for managing disputes: queue view, detail with side-by-side evidence, resolution form with justification.

## Tasks

### 1. Create admin dispute API client
In `apps/web/src/services/disputes.ts` (extend existing):
- `listAllDisputes(params)` — GET /api/v1/disputes/admin/list
- `getDisputeStats()` — GET /api/v1/disputes/admin/stats
- `resolveDispute(id, data)` — POST /api/v1/disputes/:id/resolve

### 2. Create admin dispute components
In `apps/web/src/components/features/disputes/`:

- `admin-dispute-queue.tsx` — Table/list of disputes sorted by priority:
  - Status filters (OPEN, IN_MEDIATION, all resolved)
  - Sort by: oldest first, newest, amount
  - Show: order number, buyer/seller names, reason, status, age, amount
  - Color-coded rows by urgency (OPEN > 48h = red highlight)

- `admin-dispute-detail.tsx` — Full admin view:
  - Buyer info + seller info side by side
  - Evidence from buyer (left) vs evidence from seller (right) — side by side comparison
  - Message thread
  - Order details (item, price, dates)
  - Dispute timeline

- `admin-resolution-form.tsx` — Resolution form:
  - Decision radio: Full Refund / Partial Refund / No Refund
  - If partial: refund amount input (max = order item price)
  - Justification textarea (required, min 10 chars)
  - Confirm dialog before submitting

- `admin-dispute-stats.tsx` — Stats dashboard:
  - Cards: total open, in mediation, resolved this month, avg resolution time
  - Refund totals by period

### 3. Create admin routes
- `apps/web/src/app/[locale]/(admin)/admin/disputes/page.tsx` — Admin dispute queue
- `apps/web/src/app/[locale]/(admin)/admin/disputes/[id]/page.tsx` — Admin dispute detail + resolution

### 4. Add i18n translations for admin
- Admin-specific dispute translations in PT-BR

### 5. Navigation
- Add "Disputas" to admin sidebar navigation

## Verification
- Admin sees dispute queue with filters
- Admin can view side-by-side evidence
- Admin can resolve with each decision type
- Stats show correct counts
- Resolution confirmation dialog works
