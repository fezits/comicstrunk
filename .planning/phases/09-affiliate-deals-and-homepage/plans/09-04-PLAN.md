# Plan 09-04: /deals Public Page

## Wave: 2 (depends on 09-01 + 09-02)

## Goal
Create the public /deals page with store/category filters, sort controls, deal cards, affiliate disclosure, and click tracking.

## Tasks

### 1. Create deals API client
In `apps/web/src/lib/api/deals.ts`:
- listActiveDeals(params) — GET /api/v1/deals
- getDeal(id) — GET /api/v1/deals/:id
- listStores() — GET /api/v1/deals/stores
- getClickUrl(dealId) — returns /api/v1/deals/click/:dealId (opens in new tab)
- Admin functions (for 09-06)

### 2. Create deal components
In `apps/web/src/components/features/deals/`:

- `deal-card.tsx` — Card showing: banner image (or placeholder), title, store logo+name, discount badge, coupon code (if COUPON, with copy button), expiry countdown (if expiring within 7 days), "Ver oferta" button that goes to click tracking URL (target=_blank)
- `deal-type-badge.tsx` — COUPON="Cupom" / PROMOTION="Promoção"
- `deal-filters.tsx` — Filter sidebar/bar: store select, category select, type select, sort (Mais recentes, Maior desconto, Expirando em breve)
- `deals-grid.tsx` — Responsive grid of deal cards with loading skeletons
- `affiliate-disclosure.tsx` — CONAR disclosure notice: "Este site pode receber comissão por compras realizadas através dos links de afiliados. Isso não afeta o preço que você paga."

### 3. Create /deals route page
`apps/web/src/app/[locale]/(public)/deals/page.tsx`:
- Page title "Ofertas e Cupons"
- Affiliate disclosure at top
- Filters bar
- Deals grid with pagination
- Empty state: "Nenhuma oferta disponível no momento"

### 4. Add navigation link
- Add "Ofertas" link to public navigation in nav-config.ts

## Verification
- /deals page shows active deals with filters
- Click tracking works (opens affiliate URL in new tab)
- Expired deals not shown
- Affiliate disclosure visible
- Responsive layout
