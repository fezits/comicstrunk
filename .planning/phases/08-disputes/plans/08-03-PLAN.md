# Plan 08-03: Dispute Notifications + Email Templates

## Wave: 2 (parallel with 08-02, depends on 08-01)

## Goal
Integrate dispute events with the notification system and create email templates for dispute lifecycle.

## Tasks

### 1. Create dispute email templates
In `apps/api/src/shared/email-templates/`:
- `dispute-opened.ts` — Template for seller: "Disputa aberta no pedido #XXX" with reason and description
- `dispute-response-required.ts` — Reminder to seller if no response within 24h
- `dispute-resolved.ts` — Template for both parties: resolution decision + amount if refund

### 2. Integrate notifications in dispute service
All notifications are already called in 08-01/08-02 via `createNotification()`. This plan ensures:
- Email templates are sent alongside in-app notifications
- Respect user preferences (preference gate already in notification service)
- Add email dispatch calls in the dispute service after createNotification

### 3. Add notification preference defaults
- In seed or migration: ensure DISPUTE_OPENED, DISPUTE_RESPONDED, DISPUTE_RESOLVED preference defaults exist (enabled by default)

### 4. Add 48h reminder cron (optional)
- In `shared/cron/index.ts`: Add daily job at 6 AM
- Find OPEN disputes older than 24h with no seller response
- Send reminder notification to seller
- Find OPEN disputes older than 48h with no seller response
- Auto-escalate to IN_MEDIATION status
- Send notification to admin about escalated disputes

## Verification
- Seller receives notification when dispute is opened
- Buyer receives notification when seller responds
- Both parties receive notification on resolution
- Email templates render correctly
- 48h auto-escalation works
