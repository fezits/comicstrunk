# Plan 10-01: Legal Documents API

## Wave: 1 (no dependencies)

## Goal
Legal document versioning, user acceptance tracking, mandatory re-acceptance gate, admin document management.

## Tasks

### 1. Create contracts (`packages/contracts/src/legal.ts`)
- createLegalDocumentSchema — type (enum), content (string), dateOfEffect (date), isMandatory (boolean)
- updateLegalDocumentSchema — content, dateOfEffect, isMandatory (all optional)
- acceptDocumentSchema — documentId (string)
- listLegalDocumentsSchema — paginationSchema + type filter
- LegalDocumentType enum mirror
- Export types + add to index.ts

### 2. Create legal service (`apps/api/src/modules/legal/legal.service.ts`)
- `createDocument(data)` — auto-increment version per type, create LegalDocument
- `updateDocument(id, data)` — create new version (new record, incremented version)
- `getLatestByType(type)` — latest version of each document type
- `getDocumentById(id)` — single document
- `getDocumentHistory(type)` — all versions of a document type, ordered desc
- `listDocuments(filters)` — admin: all documents paginated
- `acceptDocument(userId, documentId, ipAddress)` — record LegalAcceptance
- `hasAcceptedLatest(userId, type)` — check if user accepted latest mandatory version
- `getPendingAcceptances(userId)` — return list of mandatory documents user hasn't accepted latest version of
- `getUserAcceptances(userId)` — audit: all acceptances for a user

### 3. Create legal routes (`apps/api/src/modules/legal/legal.routes.ts`)
Public:
- `GET /latest/:type` — getLatestByType (no auth)
- `GET /:id` — getDocumentById (no auth)

Authenticated:
- `POST /accept` — acceptDocument
- `GET /pending` — getPendingAcceptances
- `GET /my-acceptances` — getUserAcceptances

Admin:
- `GET /admin/list` — listDocuments
- `GET /admin/history/:type` — getDocumentHistory
- `POST /admin` — createDocument
- `PUT /admin/:id` — updateDocument (creates new version)

### 4. Register routes in create-app.ts
- Mount at `/api/v1/legal`

### 5. Seed initial legal documents
- In seed: create v1 of TERMS_OF_USE, PRIVACY_POLICY, SELLER_TERMS (mandatory)
- And informational: PAYMENT_POLICY, RETURNS_POLICY, SHIPPING_POLICY, CANCELLATION_POLICY, COOKIES_POLICY (non-mandatory)
- All with placeholder PT-BR content

## Verification
- Admin can create/update documents (versioning works)
- Users can accept documents
- Pending acceptances detected for new mandatory versions
- Seed creates all 8 document types
