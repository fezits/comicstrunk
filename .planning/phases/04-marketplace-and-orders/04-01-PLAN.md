    - `createShippingAddressSchema`: z.object with label (optional, max 50), street (min 1, max 200), number (min 1, max 20), complement (optional, max 100), neighborhood (min 1, max 100), city (min 1, max 100), state (length 2, toUpperCase), zipCode (regex /^\d{5}-?\d{3}$/ with PT-BR error message), isDefault (boolean, default false)
    - `updateShippingAddressSchema`: createShippingAddressSchema.partial()
    - `shippingMethodSchema`: z.object with id, name, description, isActive
    - `createShippingMethodSchema`: z.object with name (min 1, max 100), description (optional, max 500), isActive (boolean, default true)

    - `updateCommissionConfigSchema`: createCommissionConfigSchema.partial()
    - `commissionPreviewSchema`: z.object({ price: z.number().positive() })

  <action>
    **apps/api/src/modules/commission/commission.service.ts:**
    - Import prisma, roundCurrency, error classes
    - `getCommissionRate(planType: string)`: Query CommissionConfig for the given planType where isActive=true; return rate as number; default 0.10 if not found
    - `calculateCommission(price: number, commissionRate: number)`: Use roundCurrency for commission = price * rate; sellerNet = price - commission; return { commission, sellerNet, rate }
    - `previewCommission(price: number, userId: string)`: Look up user's subscription to determine planType (default FREE); call getCommissionRate then calculateCommission; return { price, commissionRate, commissionAmount, sellerNet }
    - `listCommissionConfigs()`: Return all configs
    - `getCommissionConfig(id: string)`: Find by id or throw NotFoundError
    - `createCommissionConfig(data)`: Create with Prisma; validate rate is within 0-1
    - `updateCommissionConfig(id, data)`: Update existing config
    - Seed default configs if none exist: on first `getCommissionRate` call, check if configs exist; if not, create FREE=0.10 and BASIC=0.08

    **apps/api/src/modules/commission/commission.routes.ts:**
    - `GET /api/v1/commission/preview` — authenticated; query param `price` (number); calls previewCommission with req.user.userId; returns { price, commissionRate, commissionAmount, sellerNet }
    - `GET /api/v1/commission/configs` — admin only; lists all configs
    - `POST /api/v1/commission/configs` — admin only; validates createCommissionConfigSchema; creates config
    - `PUT /api/v1/commission/configs/:id` — admin only; validates updateCommissionConfigSchema; updates config
    - Use existing middleware patterns: authenticate, authorize(['ADMIN']), validate()
    - Export router with explicit `Router` type annotation (TS2742 pattern from prior phases)

    **apps/api/src/modules/marketplace/marketplace.service.ts:**
    - `searchListings(params)`: Query CollectionItems where isForSale=true AND salePrice IS NOT NULL, include catalogEntry (title, coverImageUrl, publisher, author), include user (id, displayName) as seller info. Apply optional filters: condition, price range (minPrice/maxPrice on salePrice), publisher (on catalogEntry.publisher LIKE), characterId (via catalogEntry.characters relation), seriesId (via catalogEntry.seriesId), query (LIKE on catalogEntry.title). Sort by: price (salePrice), newest (createdAt desc), condition. Paginate with page/limit. Return items + pagination metadata.
    - `getListingById(id)`: Find CollectionItem by id where isForSale=true. Include catalogEntry with full details, include user as seller. Throw NotFoundError if not found or not for sale. Return listing.
    - **IMPORTANT**: This is a PUBLIC endpoint — no authentication required. Unauthenticated visitors must be able to browse the marketplace.

    **apps/api/src/modules/marketplace/marketplace.routes.ts:**
    - `GET /` — Public (NO auth middleware). Validate query with marketplaceSearchSchema. Call searchListings(query params). Return with sendPaginated.
    - `GET /:id` — Public (NO auth middleware). Call getListingById(params.id). Return with sendSuccess.
    - Export router with `Router` type annotation.

    **apps/api/src/shared/cron/index.ts:**
    - Install node-cron: `pnpm --filter api add node-cron && pnpm --filter api add -D @types/node-cron`
    - Import `cron` from 'node-cron' and `prisma`
    - `registerCronJobs()`:
      1. Every 5 minutes (`*/5 * * * *`): Delete CartItems where expiresAt < now() — log count if > 0
      2. Daily 3 AM (`0 3 * * *`): Delete CartItems where createdAt < 7 days ago — log count if > 0
      3. Daily 4 AM (`0 4 * * *`): Update OrderItems with status PROCESSING, shippedAt null, createdAt < 7 days ago to CANCELLED with cancelledAt = now() — log count if > 0
    - Export `registerCronJobs`

    **apps/api/src/app.ts:**
    - Import `registerCronJobs` from `../shared/cron`
    - Call `registerCronJobs()` after route registration (after all app.use() for routes)
    - Import and mount commission router: `app.use('/api/v1/commission', commissionRouter)`
    - Import and mount marketplace router: `app.use('/api/v1/marketplace', marketplaceRouter)` — this is a PUBLIC route, no auth guard on the router level
