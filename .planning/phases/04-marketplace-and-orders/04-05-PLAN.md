  <name>Task 1: Create frontend API clients for cart, orders, shipping, commission, and marketplace</name>
  <files>
    apps/web/src/lib/api/marketplace.ts

    apps/web/src/lib/api/shipping.ts

  </files>
  <action>
    Follow the same pattern as existing API clients (catalog.ts, collection.ts): import apiClient, define typed functions, export.

    **apps/web/src/lib/api/marketplace.ts:**
    - `searchMarketplace(params)`: GET /api/v1/marketplace with filter params (condition, minPrice, maxPrice, publisher, characterId, query, sortBy, sortOrder, page, limit). This calls the PUBLIC marketplace endpoint created in 04-01 — no auth required, unauthenticated visitors can browse.
    - `getMarketplaceListing(id)`: GET /api/v1/marketplace/:id — gets a specific listing (public, no auth)
    - `getSellerListings(sellerId, params)`: GET /api/v1/marketplace with sellerId filter
    - `getSellerProfile(sellerId)`: GET /api/v1/users/:sellerId/profile (public profile)

    **apps/web/src/lib/api/cart.ts:**
    - `addToCart(collectionItemId)`: POST /api/v1/cart
    - `getCart()`: GET /api/v1/cart
    - `getCartSummary()`: GET /api/v1/cart/summary
    - `removeFromCart(cartItemId)`: DELETE /api/v1/cart/:id
    - `clearCart()`: DELETE /api/v1/cart

    **apps/web/src/lib/api/shipping.ts:**
    - `createAddress(data)`: POST /api/v1/shipping/addresses
    - `listAddresses()`: GET /api/v1/shipping/addresses
    - `getAddress(id)`: GET /api/v1/shipping/addresses/:id
    - `updateAddress(id, data)`: PUT /api/v1/shipping/addresses/:id
    - `deleteAddress(id)`: DELETE /api/v1/shipping/addresses/:id
    - `setDefaultAddress(id)`: PATCH /api/v1/shipping/addresses/:id/default
    - `listShippingMethods()`: GET /api/v1/shipping/methods
    - `updateTracking(orderItemId, data)`: PATCH /api/v1/shipping/tracking/:orderItemId

    **apps/web/src/lib/api/orders.ts:**
    - `createOrder(shippingAddressId)`: POST /api/v1/orders
    - `listBuyerOrders(params)`: GET /api/v1/orders/buyer
    - `listSellerOrders(params)`: GET /api/v1/orders/seller
    - `getOrder(id)`: GET /api/v1/orders/:id
    - `getOrderByNumber(orderNumber)`: GET /api/v1/orders/number/:orderNumber
    - `cancelOrder(id)`: PATCH /api/v1/orders/:id/cancel
    - `updateOrderItemStatus(itemId, status)`: PATCH /api/v1/orders/items/:itemId/status

    **apps/web/src/lib/api/commission.ts:**
    - `previewCommission(price)`: GET /api/v1/commission/preview?price=X
    - `listCommissionConfigs()`: GET /api/v1/commission/configs (admin)
    - `createCommissionConfig(data)`: POST /api/v1/commission/configs (admin)
    - `updateCommissionConfig(id, data)`: PUT /api/v1/commission/configs/:id (admin)
  </action>

    All API functions typed and consistent with existing client patterns.
  </verify>
