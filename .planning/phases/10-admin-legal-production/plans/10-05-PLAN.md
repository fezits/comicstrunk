# Plan 10-05: Legal Documents UI

## Wave: 2 (depends on 10-01)

## Goal
Legal document pages, acceptance modal, re-acceptance gate, cookie consent banner, policy pages.

## Tasks

### 1. Create legal API client (`apps/web/src/lib/api/legal.ts`)
- getLatestDocument(type) — public
- getDocumentById(id) — public
- acceptDocument(documentId) — authenticated
- getPendingAcceptances() — authenticated
- Admin functions for document management

### 2. Create legal components (`apps/web/src/components/features/legal/`)
- `legal-acceptance-modal.tsx` — Modal showing document content with "Li e aceito" button. Blocks navigation until accepted. Shows for each pending mandatory document.
- `legal-acceptance-gate.tsx` — Wrapper component: on mount, check pending acceptances. If any, show acceptance modal. Used in layout.
- `cookie-consent-banner.tsx` — Bottom banner: "Este site usa cookies..." with "Aceitar" and "Saiba mais" buttons. Persist acceptance in localStorage.
- `legal-document-page.tsx` — Full page showing document content, version, date of effect. Reusable for all document types.

### 3. Create public legal routes
- `/[locale]/(public)/terms/page.tsx` — Terms of Use
- `/[locale]/(public)/privacy/page.tsx` — Privacy Policy
- `/[locale]/(public)/policies/page.tsx` — Hub linking to all policies
- `/[locale]/(public)/policies/payment/page.tsx`
- `/[locale]/(public)/policies/returns/page.tsx`
- `/[locale]/(public)/policies/shipping/page.tsx`
- `/[locale]/(public)/policies/cancellation/page.tsx`
- `/[locale]/(public)/policies/cookies/page.tsx`
- `/[locale]/(public)/seller-terms/page.tsx` — Seller Terms

### 4. Integrate acceptance gate
- Add LegalAcceptanceGate to authenticated layout
- Add CookieConsentBanner to root layout

### 5. Add footer links
- Add links to Terms, Privacy, and Policies in footer

## Verification
- Legal document pages render content correctly
- Acceptance modal appears for pending mandatory docs
- Cookie consent banner shows once, persists
- All policy pages accessible
