1. Create `apps/web/src/components/features/checkout/pix-payment-page.tsx`:
   - Props: none (reads orderId from URL search params: `?orderId=xxx`).
   - State machine with states: 'loading', 'awaiting_payment', 'processing', 'success', 'expired', 'error'.
   - On mount (useEffect):
     a. Read `orderId` from `useSearchParams()`.
     b. If no orderId: redirect to /orders or show error.
     c. Call `initiatePayment(orderId)` to get PaymentData.
     d. If `paidAt` is already set: immediately transition to 'success'.
     e. If `pixExpiresAt` is in the past: transition to 'expired'.
     f. Otherwise: transition to 'awaiting_payment'.
   - Polling (useEffect, only when state === 'awaiting_payment'):
     a. Set interval every 5 seconds: call `getPaymentStatus(orderId)`.
     b. If `providerStatus === 'approved'` or `paidAt` is set: transition to 'success'.
     c. If `providerStatus` indicates failure: transition to 'error'.
     d. Clean up interval on unmount or state change.
   - Render by state:
     a. **loading**: Skeleton/spinner with "Gerando pagamento PIX..."
     b. **awaiting_payment**: Card with PixQrCode component, PixCountdownTimer, order summary (orderNumber, totalAmount formatted as BRL with `Intl.NumberFormat`). Instructions: "Abra o app do seu banco e escaneie o QR code ou copie o codigo PIX abaixo."
     c. **processing**: "Pagamento sendo processado..." with spinner.
     d. **success**: Green checkmark icon, "Pagamento confirmado!" message, order number, "Ver pedido" button linking to `/orders/{orderId}`. Confetti-like positive feedback.
     e. **expired**: Red X icon, "PIX expirado" message, "Tentar novamente" button that calls `initiatePayment(orderId)` to generate a new QR code.
     f. **error**: Red alert, error message, "Voltar ao pedido" button.
   - Handle timer expiry: PixCountdownTimer's `onExpired` sets state to 'expired'.
   - All text via `useTranslations('payments')`.
