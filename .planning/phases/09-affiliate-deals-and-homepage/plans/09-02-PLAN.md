# Plan 09-02: Click Tracking API

## Wave: 1 (parallel with 09-01, depends on contracts from 09-01)

## Goal
Server-side click logging with deduplication, redirect endpoint, analytics aggregation, and CSV export.

## Tasks

### 1. Create click tracking service (`apps/api/src/modules/deals/clicks.service.ts`)
- `trackClick(dealId, userId?, ipAddress, userAgent)`:
  - Validate deal exists and is active
  - Deduplication: check if same user+deal combo clicked within last 1 hour (DB query on createdAt)
  - If anonymous: use IP+deal for dedup
  - If duplicate: skip insert, still redirect
  - Create ClickLog record
  - Return the composed affiliate URL for redirect

- `getClickAnalytics(filters)`:
  - Aggregate clicks by: deal, store, category, period (day/week/month)
  - Return: totalClicks, uniqueUsers, clicksByDeal, clicksByStore, clicksByCategory
  - Support date range filter

- `exportClicksCSV(filters)`:
  - Generate CSV with columns: Deal, Store, Category, Clicks, Unique Users, Date
  - Return CSV string

### 2. Add click routes to deals.routes.ts
- `GET /click/:dealId` — trackClick + redirect (public, optionalAuthenticate)
- `GET /admin/analytics` — getClickAnalytics (admin)
- `GET /admin/analytics/export` — exportClicksCSV (admin, returns CSV)

### 3. Open-redirect protection
- In composeDealUrl: validate that the final URL domain is in the PartnerStore's baseUrl domain
- Reject any URL that doesn't match the store's registered domain

## Verification
- Click creates ClickLog and redirects to affiliate URL
- Duplicate click within 1h is skipped
- Analytics show correct aggregations
- CSV export works
- Open-redirect protection blocks non-store domains
