  <name>Task 1: Initialize monorepo root with pnpm workspaces and Turborepo</name>
  <files>
    pnpm-workspace.yaml
    turbo.json
    package.json
    tsconfig.base.json
    .npmrc
    .gitignore
    .eslintrc.json
    .prettierrc

  <action>
    **IMPORTANT:** The repo currently has `backend/`, `frontend/`, `docs/`, `git_templates/`, and `README.md` at root. Do NOT delete docs/ or git_templates/ or README.md. Remove the empty `backend/` and `frontend/` directories since the monorepo uses `apps/api` and `apps/web` instead.

    1. Create `pnpm-workspace.yaml` with packages: `['apps/*', 'packages/*']`
    2. Create root `package.json`:
       - name: "comicstrunk", private: true
       - packageManager: "pnpm@9.15.0"
       - scripts: dev ("turbo dev"), build ("turbo build"), lint ("turbo lint"), type-check ("turbo type-check")
       - devDependencies: turbo ^2.0.0, typescript ^5.0.0
    3. Create `turbo.json` with pipeline:
       - build: dependsOn ["^build"], outputs ["dist/**", ".next/**", "!.next/cache/**"]
       - dev: cache false, persistent true
       - lint: {}
       - type-check: dependsOn ["^build"]
    4. Create `tsconfig.base.json` with shared compiler options:
       - target: "ES2022", module: "ESNext", moduleResolution: "bundler"
       - strict: true, esModuleInterop: true, skipLibCheck: true
       - resolveJsonModule: true, isolatedModules: true
       - declaration: true, declarationMap: true, sourceMap: true
       - forceConsistentCasingInFileNames: true
    5. Create `.npmrc` with: `shamefully-hoist=false` and `strict-peer-dependencies=false`
    6. Create `.gitignore` with: node_modules, .next, dist, .env, .env.local, .turbo, *.tsbuildinfo, .prisma
    7. Create `.eslintrc.json` with root-level config extending "eslint:recommended" and "@typescript-eslint/recommended"
    8. Create `.prettierrc` with: semi: true, singleQuote: true, trailingComma: "all", tabWidth: 2, printWidth: 100, plugins: ["prettier-plugin-tailwindcss"]
    9. Run `pnpm install` at root to initialize the workspace
  </action>
  <verify>
    - `pnpm install` completes without errors
    - `ls apps/ packages/` shows expected directory structure targets
    - `cat pnpm-workspace.yaml` shows correct workspace config
  </verify>
  <done>
    Root monorepo configuration files exist and pnpm workspace is initialized. Turbo pipeline is defined.
  </done>
</task>

<task type="auto">
  <name>Task 2: Create apps/api, apps/web, and packages/contracts skeleton packages</name>
  <files>
    apps/api/package.json
    apps/api/tsconfig.json
    apps/api/src/app.ts
    apps/web/package.json

  </files>
  <action>
    **Create apps/api skeleton:**
    1. `apps/api/package.json`: name "api", scripts (dev: "tsx watch src/app.ts", build: "tsc", type-check: "tsc --noEmit")
    2. `apps/api/tsconfig.json`: extends "../../tsconfig.base.json", compilerOptions: { outDir: "dist", rootDir: "src" }, include: ["src"]
    3. `apps/api/src/app.ts`: minimal Express placeholder that creates http server and listens on PORT (Passenger-compatible pattern from research). Return JSON from GET /health: { status: "ok", uptime, timestamp }
    4. Install API dependencies: `pnpm --filter api add express zod cors helmet cookie-parser morgan dotenv` and dev deps: `pnpm --filter api add -D typescript @types/node @types/express @types/cors @types/morgan @types/cookie-parser tsx`

    **Create apps/web skeleton:**
    1. Use `pnpm create next-app@latest apps/web --typescript --tailwind --eslint --app --no-src-dir --import-alias "@/*"` then move to use src/ directory, OR manually scaffold with Next.js 15 App Router structure
    2. Ensure `apps/web/package.json`: name "web", depends on @comicstrunk/contracts (workspace:*)
    3. `apps/web/tsconfig.json`: extends "../../tsconfig.base.json" with Next.js-specific additions (jsx: "preserve", paths for @/*)
    4. Create minimal `src/app/layout.tsx` (html + body with lang="pt-BR") and `src/app/page.tsx` (placeholder)
    5. Install web dependencies: react, react-dom, next, tailwindcss@^3.4, etc. as listed in research

    **Create packages/contracts skeleton:**
    1. `packages/contracts/package.json`: name "@comicstrunk/contracts", main "src/index.ts", types "src/index.ts"
    2. `packages/contracts/tsconfig.json`: extends "../../tsconfig.base.json", include ["src"]
    3. `packages/contracts/src/index.ts`: export placeholder (e.g., `export const CONTRACT_VERSION = '0.1.0';`)
    4. Install: `pnpm --filter contracts add zod` and dev dep: `pnpm --filter contracts add -D typescript`

    **Wire workspace dependencies:**
    1. Add `"@comicstrunk/contracts": "workspace:*"` to both apps/api and apps/web package.json dependencies
    2. Run `pnpm install` to link workspaces
    3. Run `pnpm build` to verify the Turborepo pipeline builds all packages successfully

    **Verify import resolution:**
    - In apps/api/src/app.ts, add a test import: `import { CONTRACT_VERSION } from '@comicstrunk/contracts';` and log it in the health endpoint
    - Run `pnpm --filter api type-check` to confirm TypeScript resolves the workspace package
  </action>
