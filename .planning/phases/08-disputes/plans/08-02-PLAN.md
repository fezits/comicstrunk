# Plan 08-02: Seller Response + Admin Mediation + Refund Integration

## Wave: 2 (depends on 08-01)

## Goal
Add seller response with 48h deadline, admin mediation with resolution decisions, refund trigger, and payout hold logic.

## Tasks

### 1. Add seller response to dispute service
- `respondToDispute(disputeId, sellerId, data)`:
  - Validate dispute exists, sellerId matches dispute.sellerId
  - Validate status is OPEN
  - Check 48h deadline (warn but allow late response)
  - Create DisputeMessage with sender=seller
  - Update status to IN_MEDIATION
  - Fire notification to buyer (DISPUTE_RESPONDED)
  - Fire notification to admins (DISPUTE_RESPONDED)
  - Return updated dispute

### 2. Add admin mediation to dispute service
- `resolveDispute(disputeId, adminId, data)`:
  - Validate dispute exists, admin role
  - Validate status is OPEN or IN_MEDIATION
  - Set resolution, resolvedById, resolvedAt, refundAmount
  - Update Dispute status to chosen resolution
  - If RESOLVED_REFUND or RESOLVED_PARTIAL_REFUND:
    - Call `refundPayment()` from payments service with refundAmount
    - Update OrderItem.status to REFUNDED
  - If RESOLVED_NO_REFUND:
    - Update OrderItem.status back to previous (DELIVERED/SHIPPED)
  - Fire notification to buyer (DISPUTE_RESOLVED)
  - Fire notification to seller (DISPUTE_RESOLVED)
  - Return updated dispute

### 3. Add admin listing to dispute service
- `listAllDisputes(filters, pagination)`:
  - Admin-only: list all disputes with filters (status, date range)
  - Include buyer/seller/order info
  - Sort by createdAt desc (newest first)

- `getDisputeStats()`:
  - Count by status, average resolution time, total refunded amount

### 4. Add admin + seller routes
- `POST /:id/respond` — respondToDispute (authenticated, seller only by logic)
- `POST /:id/messages` — addMessage (authenticated, any party in dispute)
- `POST /:id/resolve` — resolveDispute (authenticated, admin only)
- `GET /admin/list` — listAllDisputes (admin only)
- `GET /admin/stats` — getDisputeStats (admin only)

### 5. Payout hold integration
- In commission service: add check for DISPUTED OrderItem status
- Skip commission payout for items in DISPUTED status
- Add comment documenting the hold logic

## Verification
- Seller can respond within 48h window
- Admin can resolve with each resolution type
- Refund triggers payment refund and updates OrderItem
- Admin sees all disputes with stats
- Commission skips disputed items
