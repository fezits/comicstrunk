<task type="auto">
  <name>Task 3: Email service with preference gate and integration hooks</name>
  <files>
    apps/api/src/modules/notifications/email.service.ts
    apps/api/src/modules/auth/auth.service.ts
    apps/api/src/modules/orders/orders.service.ts
  </files>
  <action>
1. **Create `apps/api/src/modules/notifications/email.service.ts`**:

   **Internal helper: `sendEmail(to: string, subject: string, html: string)`**
   - If !isResendConfigured(), log to console: `[EMAIL] Resend not configured, skipping: ${subject} -> ${to}` and return
   - Call `resend.emails.send({ from: RESEND_FROM_EMAIL, to, subject, html })`
   - Wrap in try/catch, log errors but never throw (fire-and-forget)

   **`sendWelcomeEmail(userId: string, email: string, userName: string)`**
   - Check isNotificationEnabled(userId, 'WELCOME')
   - If disabled, return
   - Render welcomeEmailTemplate({ userName })
   - Call sendEmail

   **`sendPaymentConfirmedEmail(userId: string, email: string, data: { userName: string; orderNumber: string; totalAmount: string; itemCount: number })`**
   - Check isNotificationEnabled(userId, 'PAYMENT_CONFIRMED')
   - Render paymentConfirmedEmailTemplate(data)
   - Call sendEmail

   **`sendOrderShippedEmail(userId: string, email: string, data: { userName: string; orderNumber: string; trackingCode: string; carrier?: string; itemTitle: string })`**
   - Check isNotificationEnabled(userId, 'ORDER_SHIPPED')
   - Render orderShippedEmailTemplate(data)
   - Call sendEmail

   **`sendItemSoldEmail(userId: string, email: string, data: { sellerName: string; orderNumber: string; itemTitle: string; salePrice: string; sellerNet: string })`**
   - Check isNotificationEnabled(userId, 'ITEM_SOLD')
   - Render itemSoldEmailTemplate(data)
   - Call sendEmail

   **`sendPasswordResetEmail(email: string, data: { userName: string; resetLink: string })`**
   - **No preference check** for password reset — always send (security-critical)
   - Render passwordResetEmailTemplate(data)
   - Call sendEmail

2. **Hook into auth module** (`apps/api/src/modules/auth/auth.service.ts`):
   - After successful signup: call `sendWelcomeEmail()` (fire-and-forget, do not await blocking)
   - In password reset request: replace existing email sending logic (if any) with `sendPasswordResetEmail()` or add the call alongside existing logic

3. **Hook into orders module** (`apps/api/src/modules/orders/orders.service.ts`):
   - In `updateOrderItemStatus()` when status becomes SHIPPED:
     - Fetch buyer's email and name
     - Fetch item title from collectionItem -> catalogEntry
     - Call `sendOrderShippedEmail()` with tracking code and carrier
   - In `createOrder()` after order creation:
     - For each unique seller, fetch seller's email and name
     - Call `sendItemSoldEmail()` for each seller with their item details

   **Note**: Payment confirmed email hook will be added when the payment webhook processes a successful payment. If the payment webhook handler already exists (from Phase 5), add the call there. If not yet built, leave a TODO comment in email.service.ts documenting where it should be called.

   **Important**: All email calls should be fire-and-forget (do not let email failures block the main operation). Use `void sendXxxEmail(...)` pattern or call after the transaction completes.
  </action>

    - With RESEND_API_KEY set: emails are sent via Resend (verify in Resend dashboard logs)
    - Password reset email is always sent regardless of notification preferences
    - Email sending failures do not crash or block the main operation
