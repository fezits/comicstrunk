1. **Create `apps/web/src/lib/api/notifications.ts`**:
   API client functions using the existing axios instance:
   - `getNotifications(params: { page: number; limit: number; unreadOnly?: boolean })` — GET /api/v1/notifications
   - `getUnreadCount()` — GET /api/v1/notifications/unread-count. Returns { count: number }
   - `markAsRead(notificationId: string)` — PATCH /api/v1/notifications/:id/read
   - `markAllAsRead()` — PATCH /api/v1/notifications/read-all
   - `getPreferences()` — GET /api/v1/notifications/preferences
   - `updatePreferences(preferences: Array<{ type: string; enabled: boolean }>)` — PUT /api/v1/notifications/preferences

2. **Create `apps/web/src/contexts/notification-context.tsx`**:
   - `NotificationProvider` wraps the app (add to layout providers)
   - Context provides: `unreadCount: number`, `refreshUnreadCount: () => void`, `isLoading: boolean`
   - On mount (if user is authenticated): fetch unread count immediately
   - Set up polling interval: call `getUnreadCount()` every 30 seconds while user is authenticated
   - Clean up interval on unmount or when user logs out
   - Uses `useAuth()` to check authentication status — only polls when authenticated
   - `refreshUnreadCount()` allows manual refresh after mark-as-read actions
   - Export `useNotifications()` hook for consuming the context
  </action>

    - NotificationProvider polls unread count when authenticated
    - Polling stops when user is not authenticated
    - API client functions match backend endpoints from 07-03
  </verify>

  </done>
