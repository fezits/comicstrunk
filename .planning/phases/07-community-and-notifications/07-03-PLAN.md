<objective>
Build the Notification System API with in-app notification endpoints (list, unread count, mark read), notification preferences CRUD, and a reusable createNotification() service function that other modules call to create notifications. Hook notification creation into existing order/shipping/payment flows.

Purpose: Provides the in-app notification backbone for the platform. Other modules call createNotification() to trigger in-app alerts. Preferences allow users to control which notification types they receive. This plan also satisfies SHIP-05 (buyer notified of shipping updates) by hooking into the order item status change flow.
Output: Working notification API endpoints, preference management, notification creation hooks in existing modules.
</objective>

<task type="auto">
  <name>Task 2: Notification service with createNotification and preference checks</name>
  <files>
    apps/api/src/modules/notifications/notifications.service.ts
  </files>
  <action>
1. **Create `apps/api/src/modules/notifications/notifications.service.ts`**:

   **`isNotificationEnabled(userId: string, type: NotificationType): Promise<boolean>`**
   - Query NotificationPreference for userId + type
   - If no row exists, return true (default enabled)
   - Otherwise return the `enabled` field value

   **`createNotification(params: { userId: string, type: NotificationType, title: string, message: string, metadata?: Record<string, unknown> })`**
   - Check if notification type is enabled for user via isNotificationEnabled()
   - If disabled, return null (skip silently)
   - Create Notification record with provided data
   - Return the created notification
   - **Important**: This function is designed to be fire-and-forget safe — callers should not await it if notification failure should not block the main operation. However, the function itself is async and returns a promise.

   **`getUserNotifications(userId, page, limit, unreadOnly?: boolean)`**
   - Fetch paginated notifications for user
   - If unreadOnly is true, filter where isRead === false
   - Order by createdAt desc
   - Return { notifications, total, page, limit }

   **`getUnreadCount(userId)`**
   - Return count of notifications where userId and isRead === false
   - Uses the [userId, isRead] composite index for performance

   **`markAsRead(userId, notificationId)`**
   - Find notification by id, verify userId matches
   - Update isRead = true, readAt = new Date()
   - Return updated notification

   **`markAllAsRead(userId)`**
   - Update all notifications where userId and isRead === false
   - Set isRead = true, readAt = new Date()
   - Return count of updated notifications

   **`getPreferences(userId)`**
   - Fetch all NotificationPreference rows for user
   - For each NotificationType in the enum, if no row exists, return { type, enabled: true } (default)
   - Return complete list of preferences for all types

   **`updatePreferences(userId, preferences: Array<{ type: NotificationType, enabled: boolean }>)`**
   - For each preference in the input array, upsert NotificationPreference (userId + type unique)
   - Return updated preferences list
  </action>
  <verify>
    - `pnpm type-check` passes
    - createNotification respects user preference settings
    - getPreferences returns all types even if no rows exist yet
  </verify>
  <done>
    Notification service handles CRUD, preference checks, and provides reusable createNotification function for other modules.
  </done>
</task>
