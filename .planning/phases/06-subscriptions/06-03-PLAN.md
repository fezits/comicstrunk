Add a new cron job to the existing `registerCronJobs()` function in `apps/api/src/shared/cron/index.ts`.

**Daily at 5 AM: Subscription reconciliation (safety net for missed webhooks)**

```
cron.schedule('0 5 * * *', async () => { ... })
```

Logic:
1. Find all subscriptions where:
   - `status` IN ('ACTIVE', 'TRIALING')
   - `planType` !== 'FREE' (only paid plans need reconciliation)
   - `currentPeriodEnd` is not null AND `currentPeriodEnd < now()` (period has expired)
   - `cancelledAt` is not null (cancellation was scheduled -- this means Stripe should have already sent `customer.subscription.deleted` but we may have missed it)

2. For each expired subscription found:
   a. If Stripe is configured (`isStripeConfigured()`), try to retrieve the Stripe subscription via `stripe.subscriptions.retrieve(stripeSubscriptionId)` and check if it's actually cancelled/expired. If Stripe says it's still active, skip it (our data may be stale).
   b. If Stripe is NOT configured (dev mode) or Stripe confirms the sub is cancelled/expired: downgrade the subscription using a `$transaction`:
      - Update subscription: status='CANCELLED', planType='FREE', cancelledAt=new Date()
      - Update user role to 'USER' (unless ADMIN)
      - Create notification: type='SUBSCRIPTION_EXPIRED', title='Assinatura expirada', message='Sua assinatura foi encerrada automaticamente. Seu plano foi revertido para FREE.'
   c. Log each downgrade: `[CRON] Downgraded subscription ${id} for user ${userId} (expired ${currentPeriodEnd})`

3. Log summary: `[CRON] Subscription reconciliation: checked N, downgraded M`

Import `stripe` and `isStripeConfigured` from shared/lib/stripe.ts at the top of the file. Wrap the entire cron handler in try/catch to prevent crashes.

Also add a note comment explaining: "This is a safety net for missed Stripe webhooks. The primary mechanism for subscription state changes is the webhook handler in stripe-webhook.service.ts."
  </action>
