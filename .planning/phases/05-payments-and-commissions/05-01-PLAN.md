<objective>
Create the PIX payment backend: Mercado Pago SDK integration for QR code generation, webhook handler with idempotency protection and signature validation, payment status polling endpoint, and contract schemas for payments and banking.

Purpose: This is the core payment infrastructure that enables buyers to pay for orders. Without it, orders remain permanently in PENDING status. The webhook handler is the automated mechanism that confirms payments.

Output: Working PIX payment API (create payment, check status), webhook endpoint that Mercado Pago can POST to, payment and banking contract schemas.
</objective>

4. Update `packages/contracts/src/index.ts`:

     - Compute HMAC-SHA256 with `process.env.MERCADOPAGO_WEBHOOK_SECRET`
     - Use `crypto.timingSafeEqual` for constant-time comparison

<task type="auto">
  <name>Task 2: Create PIX payment service, routes, and webhook handler</name>
  <files>

  <action>
1. Create `apps/api/src/modules/payments/payments.service.ts`:

   a. `initiatePixPayment(orderId: string, userId: string)`:
      - Fetch order with buyer email. Validate: order exists, buyerId === userId, status === 'PENDING'
      - Check for existing non-expired payment: if `Payment` exists for this orderId where `pixExpiresAt > now`, return it (avoid re-creating)
      - Calculate PIX expiry aligned with cart reservation: fetch the earliest cart item expiresAt for this buyer. PIX expiry = `MIN(remaining_cart_time - 5min_buffer, 30min)`. If remaining cart time < 10 minutes, throw BadRequestError('Cart reservation too short for payment')
      - If `isMercadoPagoConfigured()`: call `mpPayment.create()` with:
        - `transaction_amount: Number(order.totalAmount)`
        - `payment_method_id: 'pix'`
        - `payer: { email: buyer.email }`
        - `description: 'Comics Trunk - ${order.orderNumber}'`
        - `external_reference: orderId`
        - `date_of_expiration: pixExpiresAt.toISOString()`
        - `requestOptions: { idempotencyKey: 'pix-${orderId}' }`
      - Extract from response: `point_of_interaction.transaction_data.qr_code_base64` (pixQrCode), `qr_code` (pixCopyPaste)
      - If NOT configured (dev mode): create payment record with null pixQrCode/pixCopyPaste and providerPaymentId = `dev-${orderId}`, providerStatus = 'pending'. Log warning.
      - Upsert Payment record with Prisma: create or update using orderId-based findFirst

   b. `getPaymentStatus(orderId: string, userId: string)`:
      - Find payment by orderId. Validate order ownership (buyer check).
      - If Mercado Pago configured and providerStatus is still 'pending': fetch fresh status via `mpPayment.get({ id: providerPaymentId })`. Update providerStatus if changed. If status is 'approved', call `processPaymentConfirmation`.
      - Return payment record with current status.

   c. `processPaymentConfirmation(orderId: string)`:
      - Use `prisma.$transaction`: update order status to 'PAID', update all PENDING items to 'PAID', update Payment record paidAt and providerStatus to 'approved'.
      - This is called by both webhook handler and polling fallback.

   d. `getPaymentByOrderId(orderId: string)`:
      - Simple Prisma findFirst by orderId.

2. Create `apps/api/src/modules/payments/payments.routes.ts`:
   - All routes under authenticate middleware.
   - `POST /initiate` — validate body with `initiatePaymentSchema`, call `initiatePixPayment`, return 201 with payment data.
   - `GET /:orderId/status` — call `getPaymentStatus`, return payment data. (The `:orderId` here is the order ID, not payment ID.)

3. Create `apps/api/src/modules/payments/webhook.routes.ts`:
   - This route has NO authenticate middleware (Mercado Pago sends webhooks without JWT).
   - `POST /` — Mercado Pago webhook handler:
     a. Extract `x-signature`, `x-request-id` headers and `body.data.id`, `body.action`/`body.type`
     b. Validate signature via `validateWebhookSignature()`. If invalid and webhook secret is configured, return 401.
     c. Idempotency guard: try `prisma.webhookEvent.create({ data: { provider: 'mercadopago', eventId: String(body.id || body.data?.id), eventType: body.action || body.type || 'unknown', payload: body } })`. Catch P2002 (unique violation) -> return 200 (acknowledged but skipped).
     d. If eventType includes 'payment': fetch payment details from Mercado Pago via `mpPayment.get({ id: body.data.id })`. If status is 'approved': find Payment record by providerPaymentId, get orderId, call `processPaymentConfirmation(orderId)`.
     e. Mark webhook as processed: `webhookEvent.update({ processedAt: new Date() })`.
     f. Always return 200 (even on errors, log but don't fail — MP will retry on non-200).

4. Update `apps/api/src/create-app.ts`:
   - Import `paymentsRoutes` from `./modules/payments/payments.routes`
   - Import `webhookRoutes` from `./modules/payments/webhook.routes`
   - Mount: `app.use('/api/v1/payments', paymentsRoutes);`
   - Mount: `app.use('/api/v1/webhooks/mercadopago', webhookRoutes);`
   - IMPORTANT: Webhook route does NOT need raw body (unlike Stripe). The MP signature uses parsed data.id, not raw body. Standard `express.json()` is fine.
  </action>

    - apps/api/src/modules/payments/payments.routes.ts exists with POST /initiate and GET /:orderId/status
  </verify>
