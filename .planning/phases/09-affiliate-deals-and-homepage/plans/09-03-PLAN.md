# Plan 09-03: Homepage Configuration API

## Wave: 1 (parallel with 09-01 + 09-02)

## Goal
Admin-configurable homepage sections with CRUD, ordering, and data assembly endpoint.

## Tasks

### 1. Add homepage schemas to contracts (deals.ts)
- createHomepageSectionSchema — type (enum), title (optional), sortOrder (number), isVisible (boolean), contentRefs (object with dealIds/catalogIds arrays)
- updateHomepageSectionSchema — all optional

### 2. Create homepage service (`apps/api/src/modules/homepage/homepage.service.ts`)
- `listSections()` — admin: all sections sorted by sortOrder
- `createSection(data)` — create new section
- `updateSection(id, data)` — update section (title, order, visibility, content refs)
- `deleteSection(id)` — remove section
- `reorderSections(orderedIds)` — bulk update sortOrder based on array position

- `getHomepageData()` — public endpoint:
  - Fetch all visible sections, sorted by sortOrder
  - For each section type, resolve content:
    - BANNER_CAROUSEL: fetch referenced deals/catalog entries with banner images
    - CATALOG_HIGHLIGHTS: fetch referenced catalog entries with ratings
    - DEALS_OF_DAY: fetch active deals expiring soonest (or referenced dealIds)
    - FEATURED_COUPONS: fetch active COUPON type deals (or referenced dealIds)
  - Return assembled sections with resolved data

### 3. Create homepage routes (`apps/api/src/modules/homepage/homepage.routes.ts`)
Public:
- `GET /` — getHomepageData (no auth)

Admin:
- `GET /admin/sections` — listSections
- `POST /admin/sections` — createSection
- `PUT /admin/sections/:id` — updateSection
- `DELETE /admin/sections/:id` — deleteSection
- `POST /admin/sections/reorder` — reorderSections

### 4. Register routes in create-app.ts
- Mount at `/api/v1/homepage`

### 5. Seed homepage sections
- Add to prisma/seed.ts: 4 default sections (BANNER_CAROUSEL, CATALOG_HIGHLIGHTS, DEALS_OF_DAY, FEATURED_COUPONS) all visible, ordered 1-4

## Verification
- Admin can create/edit/delete/reorder sections
- Public endpoint returns assembled homepage data with resolved references
- Only visible sections appear in public endpoint
