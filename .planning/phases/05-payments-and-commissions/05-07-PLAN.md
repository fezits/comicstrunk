2. Create `apps/web/src/components/features/orders/payment-status-section.tsx`:
   - Props: `{ orderId: string; orderStatus: string }`.
   - Client component that fetches payment data on mount:
     - Call `getPaymentStatus(orderId)` from `lib/api/payments.ts`.
     - Handle errors gracefully (payment may not exist for very old orders).
   - Display as a shadcn Card with CreditCard icon in title:
     a. **No payment record found**: Show "Pagamento nao iniciado" with a "Pagar com PIX" button that links to `/checkout/payment?orderId=${orderId}`. Only show button if orderStatus === 'PENDING'.
     b. **Payment pending (providerStatus === 'pending')**: Show "Aguardando pagamento PIX" in yellow badge. Show "Ir para pagamento" button linking to `/checkout/payment?orderId=${orderId}`. Show PIX expiry countdown if pixExpiresAt exists and is in the future.
     c. **Payment approved (paidAt is set)**: Show "Pagamento confirmado" in green badge. Show payment date (paidAt formatted as DD/MM/YYYY HH:MM). Show amount formatted as BRL.
     d. **Payment refunded (refundedAmount > 0)**: Show "Reembolso: R$ XX,XX" in blue badge. If partial: show original amount and refunded amount.
     e. **Payment expired (pixExpiresAt in the past and not paid)**: Show "PIX expirado" in red badge. Show "Tentar novamente" button linking to payment page.
   - Use `useTranslations('orders')` for strings (extend existing orders translations).
