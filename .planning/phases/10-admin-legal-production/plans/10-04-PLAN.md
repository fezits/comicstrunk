# Plan 10-04: Admin Dashboard Metrics + User Management API

## Wave: 1 (parallel)

## Goal
Aggregated dashboard metrics, admin user management (list, search, suspend, plan adjustment).

## Tasks

### 1. Create contracts (`packages/contracts/src/admin.ts`)
- listUsersSchema — paginationSchema + search (string), role filter, status filter
- updateUserRoleSchema — role (enum)
- suspendUserSchema — reason (string)
- Export types + add to index.ts

### 2. Create admin dashboard service (`apps/api/src/modules/admin/admin.service.ts`)
- `getDashboardMetrics()`:
  - Total users count
  - New users this month
  - Total orders count + orders today
  - Revenue total (sum of paid payments)
  - Revenue this month
  - Catalog entries count (approved)
  - Pending catalog approvals count
  - Active disputes count
  - Active subscriptions count
  - Contact messages unread count

- `listUsers(filters)` — paginated, searchable by name/email, filterable by role
- `getUser(id)` — full user detail with stats (orders, collection size, reviews, subscription)
- `updateUserRole(id, role)` — change user role (cannot change own role)
- `suspendUser(id, reason)` — set role to USER, clear subscription, add suspension note
- `unsuspendUser(id)` — restore user

### 3. Create admin routes (`apps/api/src/modules/admin/admin.routes.ts`)
Admin only:
- `GET /dashboard` — getDashboardMetrics
- `GET /users` — listUsers
- `GET /users/:id` — getUser
- `PUT /users/:id/role` — updateUserRole
- `POST /users/:id/suspend` — suspendUser
- `POST /users/:id/unsuspend` — unsuspendUser

### 4. Register routes in create-app.ts
- Mount at `/api/v1/admin`

## Verification
- Dashboard returns correct aggregated metrics
- User list with search and filters
- Role change works (cannot change own)
- Suspend/unsuspend works
