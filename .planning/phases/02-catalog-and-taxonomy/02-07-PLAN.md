**Admin Catalog API Service (`apps/web/src/lib/api/admin-catalog.ts`):**
- `getAdminCatalogList(params: { page, limit, approvalStatus? })`: GET `/api/v1/catalog/admin/list` with auth header
- `createCatalogEntry(data: FormData)`: POST `/api/v1/catalog` with JSON body (for metadata). Cover image uploaded separately.
- `updateCatalogEntry(id: string, data)`: PUT `/api/v1/catalog/${id}`
- `deleteCatalogEntry(id: string)`: DELETE `/api/v1/catalog/${id}`
- `submitForReview(id: string)`: PATCH `/api/v1/catalog/${id}/submit`
- `approveCatalogEntry(id: string)`: PATCH `/api/v1/catalog/${id}/approve`
- `rejectCatalogEntry(id: string, reason: string)`: PATCH `/api/v1/catalog/${id}/reject` with body `{ rejectionReason: reason }`
- `uploadCoverImage(id: string, file: File)`: POST `/api/v1/catalog/${id}/cover` as FormData
- `importCSV(file: File)`: POST `/api/v1/catalog/import` as FormData. Return `{ created, errors, total }`.
- `exportCSV()`: GET `/api/v1/catalog/export` with responseType 'blob'. Trigger download.

**Approval Badge Component (`apps/web/src/components/features/catalog/approval-badge.tsx`):**
- Props: `status: 'DRAFT' | 'PENDING' | 'APPROVED' | 'REJECTED'`
- Badge colors: DRAFT = gray/outline, PENDING = yellow/warning, APPROVED = green/success, REJECTED = red/destructive
- Display translated status text

**Catalog Form Component (`apps/web/src/components/features/catalog/catalog-form.tsx`):**
- Props: `entry?: CatalogEntry` (for edit mode), `onSubmit: (data) => void`, `onCancel: () => void`
- Use react-hook-form with zodResolver and createCatalogEntrySchema from contracts
- Fields: title (Input, required), author (Input), publisher (Input), imprint (Input), barcode (Input), isbn (Input), description (Textarea), series (Select dropdown — fetch series list), volumeNumber (Input number, shown when series selected), editionNumber (Input number, shown when series selected), categories (multi-select checkboxes — fetch categories), tags (multi-select checkboxes — fetch tags), characters (multi-select checkboxes — fetch characters)
- Cover image: File input with preview (show current coverImageUrl in edit mode)
- Submit button: "Criar" (create mode) or "Salvar" (edit mode)
- Fetch taxonomy lists (series, categories, tags, characters) on mount for dropdowns

**Admin Catalog Page (`apps/web/src/app/[locale]/(admin)/admin/catalog/page.tsx`):**
- Tab layout (shadcn Tabs): "Pendentes" (PENDING), "Rascunhos" (DRAFT), "Aprovados" (APPROVED), "Rejeitados" (REJECTED), "Todos"
- Each tab fetches entries with appropriate approvalStatus filter
- Table layout: cover thumbnail (40x60), title, author, publisher, approval badge, date, actions
- Actions per entry:
  - DRAFT: "Enviar para revisao" button, Edit link, Delete button
  - PENDING: "Aprovar" button (green), "Rejeitar" button (red, opens modal for reason), Edit link
  - APPROVED: "Editar" link
  - REJECTED: shows rejection reason tooltip, "Editar" link
- Approve: call approveCatalogEntry, refresh list
- Reject: open dialog with Textarea for reason, call rejectCatalogEntry
- Delete: confirmation dialog ("Tem certeza?"), call deleteCatalogEntry
- "Novo Catalogo" button at top linking to /admin/catalog/new
- "Importar CSV" button linking to /admin/catalog/import
- "Exportar CSV" button that triggers exportCSV download
- Pagination at bottom

**New Catalog Entry Page (`apps/web/src/app/[locale]/(admin)/admin/catalog/new/page.tsx`):**
- Render CatalogForm in create mode
- On submit: call createCatalogEntry, then if cover image provided call uploadCoverImage, then redirect to /admin/catalog

**Edit Catalog Entry Page (`apps/web/src/app/[locale]/(admin)/admin/catalog/[id]/edit/page.tsx`):**

**CSV Import Page (`apps/web/src/app/[locale]/(admin)/admin/catalog/import/page.tsx`):**
