  <files>

  <action>
1. **Create `apps/web/src/lib/api/favorites.ts`**:
   API client functions using the existing axios instance:
   - `toggleFavorite(catalogEntryId: string)` — POST /api/v1/favorites/toggle, body: { catalogEntryId }. Returns { favorited: boolean }
   - `getUserFavorites(params: { page: number; limit: number })` — GET /api/v1/favorites. Returns paginated favorites with catalog entry data.
   - `checkIsFavorited(catalogEntryId: string)` — GET /api/v1/favorites/check/:catalogEntryId. Returns { isFavorited: boolean }

2. **Create `apps/web/src/components/features/favorites/favorite-button.tsx`**:
   - Props: `catalogEntryId: string`, `initialFavorited?: boolean`, `size?: 'sm' | 'md'`, `className?: string`
   - Uses Heart icon from lucide-react
   - State: `isFavorited` (boolean), initialized from `initialFavorited` prop
   - On mount: if `initialFavorited` not provided, call `checkIsFavorited()` to get current state (only if user is authenticated)
   - On click:
     - If not authenticated: redirect to login page (use router.push)
     - If authenticated: optimistic update (toggle isFavorited immediately), then call toggleFavorite API
     - On error: revert optimistic update, show toast error
   - Visual:
     - Favorited: filled red heart (fill="currentColor" className="text-red-500")
     - Not favorited: outline heart (text-muted-foreground, hover:text-red-500)
   - Accessible: aria-label "Adicionar aos favoritos" / "Remover dos favoritos"
   - Prevent event bubbling (stopPropagation) so clicking the heart on a card doesn't navigate to detail page
   - Size variants: sm=16px icon, md=20px icon
  </action>
  <verify>
    - `pnpm type-check` passes
    - FavoriteButton toggles visual state on click
    - FavoriteButton handles unauthenticated users by redirecting to login
    - API client functions match backend endpoints from 07-02
  </verify>
  <done>
    FavoriteButton component and API client ready with optimistic updates and auth handling.
  </done>
