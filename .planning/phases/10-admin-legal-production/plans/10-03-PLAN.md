# Plan 10-03: Contact Form API

## Wave: 1 (parallel)

## Goal
Public contact form with rate limiting, admin management panel.

## Tasks

### 1. Create contracts (`packages/contracts/src/contact.ts`)
- createContactMessageSchema — name, email, category (enum), subject, message
- listContactMessagesSchema — paginationSchema + isRead, isResolved, category filters
- ContactCategory enum
- Export types + add to index.ts

### 2. Create contact service (`apps/api/src/modules/contact/contact.service.ts`)
- `submitMessage(data)` — create ContactMessage
- `listMessages(filters)` — admin: paginated, filterable by read/resolved/category
- `getMessage(id)` — single message
- `markAsRead(id)` — set isRead=true, readAt=now
- `markAsResolved(id)` — set isResolved=true, resolvedAt=now
- `deleteMessage(id)` — hard delete

### 3. Create contact routes (`apps/api/src/modules/contact/contact.routes.ts`)
Public:
- `POST /` — submitMessage (with rate limiter: max 5 per IP per hour)

Admin:
- `GET /admin/list` — listMessages
- `GET /admin/:id` — getMessage
- `PUT /admin/:id/read` — markAsRead
- `PUT /admin/:id/resolve` — markAsResolved
- `DELETE /admin/:id` — deleteMessage

### 4. Rate limiter
- Use express-rate-limit on the POST endpoint
- 5 submissions per IP per hour
- Custom message: "Muitas mensagens enviadas. Tente novamente em uma hora."

### 5. Register routes in create-app.ts
- Mount at `/api/v1/contact`

## Verification
- Public can submit contact form
- Rate limiter blocks after 5 submissions/hour
- Admin can list, read, resolve, delete messages
