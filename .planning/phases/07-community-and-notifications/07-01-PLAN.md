<task type="auto">
  <name>Task 2: Reviews service with rating aggregation</name>
  <files>
    apps/api/src/modules/reviews/reviews.service.ts
  </files>
  <action>
1. **Create `apps/api/src/modules/reviews/reviews.service.ts`**:

   **Helper: `recalculateCatalogRating(catalogEntryId: string)`**
   - Run `prisma.review.aggregate({ where: { catalogEntryId }, _avg: { rating: true }, _count: { rating: true } })`
   - Update `prisma.catalogEntry.update({ where: { id: catalogEntryId }, data: { averageRating: avg || 0, ratingCount: count } })`
   - Wrap in a transaction for atomicity

   **`createCatalogReview(userId, data: CreateCatalogReviewInput)`**
   - Verify catalogEntryId exists and is APPROVED
   - Create review with userId, catalogEntryId, rating, text
   - The @@unique([userId, catalogEntryId]) constraint handles duplicate prevention â€” catch Prisma P2002 and throw ConflictError('You have already reviewed this catalog entry')
   - Call `recalculateCatalogRating(catalogEntryId)` after creation
   - Return the created review with user info

   **`createSellerReview(userId, data: CreateSellerReviewInput)`**
   - Fetch the order by orderId, verify it exists
   - Verify order.buyerId === userId (only buyer can review seller)
   - Verify order.status === 'COMPLETED'
   - Verify data.sellerId appears in order.orderItems (at least one item with that sellerId)
   - Create review with userId, sellerId, orderId, rating, text
   - The @@unique([userId, sellerId, orderId]) constraint prevents duplicates â€” catch P2002 and throw ConflictError
   - Return the created review

   **`updateReview(userId, reviewId, data: UpdateReviewInput)`**
   - Find review by id, verify it exists
   - Verify review.userId === userId (only author can edit)
   - Update rating and/or text fields
   - If the review has a catalogEntryId (catalog review), call `recalculateCatalogRating`
   - Return updated review

   **`deleteReview(userId, reviewId)`**
   - Find review by id, verify it exists
   - Verify review.userId === userId (only author can delete)
   - Delete the review
   - If catalog review, call `recalculateCatalogRating`

   **`getCatalogReviews(catalogEntryId, page, limit)`**
   - Fetch paginated reviews where catalogEntryId matches
   - Include user name and avatar
   - Order by createdAt desc
   - Return { reviews, total, page, limit }

   **`getSellerReviews(sellerId, page, limit)`**
   - Fetch paginated reviews where sellerId matches
   - Include user name and avatar
   - Order by createdAt desc
   - Also calculate seller average: `prisma.review.aggregate({ where: { sellerId }, _avg: { rating: true }, _count: { rating: true } })`
   - Return { reviews, total, page, limit, averageRating, ratingCount }

   **`getSellerAverageRating(sellerId)`**
   - Return { averageRating, ratingCount } for a seller
   - Used by seller profile pages

   **`getUserReviewForCatalog(userId, catalogEntryId)`**
   - Return the user's existing review for a catalog entry (or null)
   - Used by frontend to show edit form vs. create form
  </action>
  <verify>
    - `pnpm type-check` passes
    - Service functions handle all edge cases: non-existent catalog entry, non-COMPLETED order, wrong buyer, duplicate review
  </verify>
  <done>
    Reviews service handles catalog and seller review CRUD with atomic rating aggregation.
  </done>
