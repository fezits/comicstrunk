  <files>

  <action>
    **Auth context (`src/lib/auth/auth-context.tsx`):**
    - Create React context with:
      ```typescript
      interface AuthContextType {
        user: AuthUser | null;
        isLoading: boolean;
        isAuthenticated: boolean;
        login: (email: string, password: string) => Promise<void>;
        signup: (input: SignupInput) => Promise<void>;
        logout: () => Promise<void>;
        refreshSession: () => Promise<void>;
      }

      interface AuthUser {
        id: string;
        name: string;
        email: string;
        role: 'USER' | 'SUBSCRIBER' | 'ADMIN';
      }
      ```

    **Auth provider (`src/lib/auth/auth-provider.tsx`):**
    - Client component ("use client")
    - On mount: attempt silent refresh (POST /auth/refresh) to restore session
      - If refresh succeeds: set user and access token (via setAccessToken from api client)
      - If refresh fails: user stays null (not logged in)
      - Set isLoading=false after attempt completes
    - `login(email, password)`: POST /auth/login → set access token + user state → navigate to /
    - `signup(input)`: POST /auth/signup → set access token + user state → navigate to /
    - `logout()`: POST /auth/logout → clear access token + user state → navigate to /login
    - `refreshSession()`: POST /auth/refresh → update access token + user state
    - Wrap children with AuthContext.Provider

    **useAuth hook (`src/lib/auth/use-auth.ts`):**
    - `useAuth()` hook that consumes AuthContext, throws if used outside provider

    **Wire into locale layout (`src/app/[locale]/layout.tsx`):**
    - Add AuthProvider wrapping page content (inside ThemeProvider and NextIntlClientProvider)
    - Order: NextIntlClientProvider → ThemeProvider → AuthProvider → {children}
  </action>
  <verify>
    - `pnpm --filter web type-check` passes
    - `pnpm --filter web dev` starts — no errors in console
    - AuthProvider attempts refresh on page load (visible in network tab)
    - useAuth() is usable in any client component
  </verify>
  <done>
    Auth context and provider exist. On page load, the provider silently tries to refresh the session. Login, signup, logout, and refresh methods are available to all components via useAuth().
