    - darkMode: "class"

    - Pin to tailwindcss@^3.4 (NOT v4 per research)

    **PostCSS config (`postcss.config.js`):**
    - tailwindcss and autoprefixer plugins

    **Global styles (`src/styles/globals.css`):**
    Per the CONTEXT.md locked decisions and research Pattern 5:
    - @tailwind base/components/utilities directives
    - Custom CSS variables in @layer base for both :root (light) and .dark (dark) themes
    - Purple dominant: `--primary: 263 84% 55%` (maps to #7C3AED)
    - Blue secondary: `--secondary: 217 91% 60%`
    - Dark theme: deep backgrounds (`--background: 240 10% 3.9%`), collector's vault feel
    - Light theme: clean whites with purple accents
    - All standard shadcn/ui variables (card, popover, muted, accent, destructive, border, input, ring)
    - Custom gradient utility classes for purple-to-blue gradients on buttons/headers per CONTEXT.md

    **next-intl setup:**
    1. `src/i18n/routing.ts`: `defineRouting({ locales: ['pt-BR'], defaultLocale: 'pt-BR' })`
    2. `src/i18n/request.ts`: `getRequestConfig` that loads messages for the requested locale
    3. `src/middleware.ts`: `createMiddleware(routing)` with matcher config excluding api, _next, static
    4. `src/messages/pt-BR.json`: Initial PT-BR messages for:
       - common: { appName, loading, error, success, save, cancel, back, search, noResults }
       - nav: { home, collection, marketplace, deals, settings, admin, login, signup, logout }
       - auth: { login, signup, email, password, name, forgotPassword, resetPassword, acceptTerms, ... }
       - theme: { dark, light, toggle }
       - errors: { required, invalidEmail, passwordTooShort, ... }
    5. Update `next.config.ts` with next-intl plugin: `createNextIntlPlugin()(config)` plus standalone output and outputFileTracingRoot

    - Import globals.css

    - This layout is the shell all pages render inside

    - Same navigation structure as Sidebar
    - Triggered by hamburger button in Header
    - Closes on navigation

    **Route group layouts:**
    - `(public)/layout.tsx`: header + sidebar layout (sidebar + main content area)
    - `(public)/page.tsx`: simple landing page placeholder with "Welcome to Comics Trunk" in PT-BR, styled with the dark immersive vibe
    - `(auth)/layout.tsx`: centered card layout per CONTEXT.md ("centered card for public/auth pages") — no sidebar, just centered content with background
    - `(collector)/layout.tsx`: header + sidebar layout (reuse same shell — will be populated in Phase 3)
    - `(seller)/layout.tsx`: header + sidebar layout (future Phase 4)
    - `(orders)/layout.tsx`: header + sidebar layout (future Phase 4)
    - `(admin)/layout.tsx`: header + sidebar layout with admin-specific nav (future Phase 10)

    **Responsive breakpoints per INFRA-08:**
    - Mobile: < 768px (no sidebar, hamburger menu, stacked layout)
    - Tablet: 768-1023px (no sidebar, hamburger menu, wider content)
    - Desktop: 1024px+ (sidebar visible, full layout)

    **API client (`src/lib/api/client.ts`):**
    Follow the exact pattern from research Code Examples section:
    - Axios instance with baseURL from NEXT_PUBLIC_API_URL
    - withCredentials: true (for httpOnly cookie refresh)
    - In-memory access token variable
    - Request interceptor: attach Bearer token
    - Response interceptor: on 401, coordinate single refresh attempt (avoid race condition)
    - If refresh fails: clear token, redirect to /pt-BR/login
    - Export: apiClient, setAccessToken, getAccessToken
