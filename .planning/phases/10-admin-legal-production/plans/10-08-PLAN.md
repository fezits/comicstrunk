# Plan 10-08: Unified Admin Dashboard + User/Legal/Contact Management UI

## Wave: 2 (depends on 10-01 + 10-02 + 10-03 + 10-04)

## Goal
Admin home with KPI cards, user management, legal document editor, LGPD request management, contact messages panel.

## Tasks

### 1. Create admin API clients
In `apps/web/src/lib/api/admin.ts`:
- getDashboardMetrics()
- listUsers(params)
- getUser(id)
- updateUserRole(id, role)
- suspendUser(id, reason)
- unsuspendUser(id)

Extend `apps/web/src/lib/api/legal.ts` with admin:
- listDocuments(params)
- createDocument(data)
- getDocumentHistory(type)

Extend `apps/web/src/lib/api/lgpd.ts` with admin:
- listAllRequests(params)
- processRequest(id)
- completeRequest(id)
- rejectRequest(id, reason)

Create `apps/web/src/lib/api/admin-contact.ts`:
- listMessages(params)
- getMessage(id)
- markAsRead(id)
- markAsResolved(id)
- deleteMessage(id)

### 2. Create admin dashboard components (`apps/web/src/components/features/admin/`)
- `admin-dashboard.tsx` — KPI cards grid:
  - Usuários totais (Users icon, blue)
  - Pedidos hoje (ShoppingCart icon, green)
  - Receita do mês (DollarSign icon, purple)
  - Catálogo (Library icon, orange)
  - Aprovações pendentes (Clock icon, yellow)
  - Disputas ativas (Scale icon, red)
  - Assinaturas ativas (Crown icon, green)
  - Mensagens não lidas (Mail icon, blue)
  - Quick action links below

- `admin-user-management.tsx` — User table:
  - Columns: Nome, Email, Plano, Data cadastro, Status
  - Search by name/email
  - Filter by role (USER, SUBSCRIBER, ADMIN)
  - Actions: view detail, change role, suspend/unsuspend
  - Confirmation dialogs for destructive actions

- `admin-user-detail.tsx` — User detail page:
  - Profile info, plan info, stats (orders, collection, reviews)
  - Role change dropdown
  - Suspend/unsuspend button

### 3. Create admin legal components
- `admin-legal-documents.tsx` — Document management:
  - List documents by type with latest version number
  - Create new document: type select, content textarea (rich text or plain), dateOfEffect, mandatory toggle
  - View version history for each type
  - Each version shows: version number, date, first 100 chars preview

### 4. Create admin LGPD components
- `admin-lgpd-requests.tsx` — Data request management:
  - Table: user, type (ACCESS/CORRECTION/DELETION/EXPORT), status, date
  - Filter by status (PENDING, PROCESSING, COMPLETED, REJECTED)
  - Actions: process, complete, reject (with reason dialog)
  - Highlight DELETION requests in red

### 5. Create admin contact components
- `admin-contact-messages.tsx` — Contact messages panel:
  - Table: nome, email, categoria, assunto, data, status (lida/resolvida)
  - Filter by read/resolved/category
  - Click to expand/view full message
  - Actions: mark read, mark resolved, delete with confirmation

### 6. Create/update admin route pages
- Update `apps/web/src/app/[locale]/(admin)/admin/page.tsx` — Admin dashboard (replace or create)
- Create `apps/web/src/app/[locale]/(admin)/admin/users/page.tsx` — User management
- Create `apps/web/src/app/[locale]/(admin)/admin/users/[id]/page.tsx` — User detail
- Create `apps/web/src/app/[locale]/(admin)/admin/legal/page.tsx` — Legal document management
- Create `apps/web/src/app/[locale]/(admin)/admin/lgpd/page.tsx` — LGPD requests
- Create `apps/web/src/app/[locale]/(admin)/admin/contact/page.tsx` — Contact messages

### 7. Update admin navigation
- Add: Legal, LGPD, Contato to admin nav in nav-config.ts
- Add PT-BR translations

## Verification
- Dashboard shows correct KPI metrics
- User management: search, filter, role change, suspend
- Legal documents: create, view history, version numbers
- LGPD requests: list, process, complete, reject
- Contact messages: list, read, resolve, delete
