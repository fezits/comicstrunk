# Plan 08-04: Buyer/Seller Dispute UI

## Wave: 3 (depends on 08-01 + 08-02)

## Goal
Create the frontend pages for buyers to open and track disputes, and sellers to respond and view dispute history.

## Tasks

### 1. Create dispute API client service
In `apps/web/src/services/disputes.ts`:
- `createDispute(data)` — POST /api/v1/disputes (multipart for evidence)
- `getDispute(id)` — GET /api/v1/disputes/:id
- `listBuyerDisputes(params)` — GET /api/v1/disputes/my/buyer
- `listSellerDisputes(params)` — GET /api/v1/disputes/my/seller
- `respondToDispute(id, data)` — POST /api/v1/disputes/:id/respond
- `addEvidence(id, file, description)` — POST /api/v1/disputes/:id/evidence
- `addMessage(id, message)` — POST /api/v1/disputes/:id/messages
- `cancelDispute(id)` — POST /api/v1/disputes/:id/cancel

### 2. Create dispute components
In `apps/web/src/components/features/disputes/`:

- `dispute-status-badge.tsx` — Badge with color per status (OPEN=yellow, IN_MEDIATION=blue, RESOLVED_REFUND=green, RESOLVED_NO_REFUND=red, CANCELLED=gray)
- `dispute-reason-badge.tsx` — Human-readable reason labels in PT-BR
- `dispute-timeline.tsx` — Vertical timeline showing dispute lifecycle events (opened, seller responded, admin resolved) with timestamps
- `dispute-message-thread.tsx` — Chat-style thread showing messages from buyer, seller, admin with avatars/roles
- `evidence-gallery.tsx` — Grid of uploaded evidence images with lightbox/modal view
- `evidence-upload-form.tsx` — Dropzone for image upload with description field, max 5 files
- `create-dispute-form.tsx` — Form: select reason (dropdown), description (textarea), upload evidence, submit
- `dispute-response-form.tsx` — Seller response: message textarea + evidence upload
- `dispute-detail-page.tsx` — Full dispute view: status badge, timeline, messages, evidence from both parties, actions (respond/cancel)
- `buyer-disputes-page.tsx` — List of buyer's disputes with filters (status), pagination, link to detail
- `seller-disputes-page.tsx` — List of seller's disputes with filters (status), pagination, link to detail

### 3. Create routes
- `apps/web/src/app/[locale]/(orders)/disputes/page.tsx` — Buyer disputes list
- `apps/web/src/app/[locale]/(orders)/disputes/[id]/page.tsx` — Dispute detail (buyer view)
- `apps/web/src/app/[locale]/(orders)/disputes/new/page.tsx` — Create dispute (with orderItemId query param)
- `apps/web/src/app/[locale]/(seller)/seller/disputes/page.tsx` — Seller disputes list
- `apps/web/src/app/[locale]/(seller)/seller/disputes/[id]/page.tsx` — Seller dispute detail with response form

### 4. Add i18n translations
- Add dispute-related translations to PT-BR locale files

### 5. Navigation integration
- Add "Disputas" link in buyer order detail when status allows
- Add "Disputas" link in seller order detail
- Add disputes to sidebar/navigation menus

## Verification
- Buyer can open dispute from order detail
- Buyer can view dispute list and detail
- Seller can view dispute and respond with evidence
- Status badges show correct colors
- Timeline shows complete lifecycle
- Evidence images display correctly
