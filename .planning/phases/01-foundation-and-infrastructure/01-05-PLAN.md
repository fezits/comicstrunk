    **Common types (`src/common.ts`):**
    ```typescript
    import { z } from 'zod';

    // === Enums (single source of truth) ===
    export const UserRole = {
      USER: 'USER',
      SUBSCRIBER: 'SUBSCRIBER',
      ADMIN: 'ADMIN',
    } as const;
    export type UserRole = (typeof UserRole)[keyof typeof UserRole];

    // === API Response wrappers ===
    export interface ApiSuccessResponse<T> {
      success: true;
      data: T;
    }

    export interface ApiErrorResponse {
      success: false;
      error: {
        message: string;
        code?: string;
        details?: unknown;
      };
    }

    export type ApiResponse<T> = ApiSuccessResponse<T> | ApiErrorResponse;

    // === Pagination ===
    export const paginationSchema = z.object({
      page: z.coerce.number().int().positive().default(1),
      limit: z.coerce.number().int().positive().max(100).default(20),
    });

    export type PaginationInput = z.infer<typeof paginationSchema>;

    export interface PaginatedResponse<T> {
      success: true;
      data: T[];
      pagination: {
        page: number;
        limit: number;
        total: number;
        totalPages: number;
      };
    }
    ```

    **Auth contracts (`src/auth.ts`):**
    Follow the exact pattern from 01-RESEARCH.md Code Examples section:

    - `signupSchema`: name (2-100 chars), email (valid email), password (8+ chars, 1 uppercase, 1 lowercase, 1 number), acceptedTerms (literal true with custom error "Must accept terms")
    - `loginSchema`: email (valid email), password (min 1 char)
    - `resetPasswordRequestSchema`: email (valid email)
    - `resetPasswordConfirmSchema`: token (min 1 char), password (same rules as signup)
    - Inferred types: SignupInput, LoginInput, ResetPasswordRequestInput, ResetPasswordConfirmInput
    - Response interfaces: AuthResponse (accessToken + user { id, name, email, role }), MessageResponse

    **User contracts (`src/users.ts`):**
    - `updateProfileSchema`: name (optional, 2-100), bio (optional, max 500)
    - `UserProfile` interface: id, email, name, role, avatarUrl (nullable), bio (nullable), createdAt
    - Inferred type: UpdateProfileInput

    **Barrel export (`src/index.ts`):**
    - `export * from './auth';`
    - `export * from './users';`
    - `export * from './common';`
