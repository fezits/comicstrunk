**Add to catalog.service.ts — `searchCatalog(filters: CatalogSearchInput)`:**

Build a dynamic Prisma `where` clause that ALWAYS includes `approvalStatus: 'APPROVED'` (public search only shows approved entries):

```typescript
const where: Prisma.CatalogEntryWhereInput = {
  approvalStatus: 'APPROVED',
};

if (filters.title) {
  where.title = { contains: filters.title };
}
if (filters.publisher) {
  where.publisher = { contains: filters.publisher };
}
if (filters.seriesId) {
  where.seriesId = filters.seriesId;
}
if (filters.categoryIds?.length) {
  where.categories = { some: { categoryId: { in: filters.categoryIds } } };
}
if (filters.characterIds?.length) {
  where.characters = { some: { characterId: { in: filters.characterIds } } };
}
if (filters.tagIds?.length) {
  where.tags = { some: { tagId: { in: filters.tagIds } } };
}
if (filters.yearFrom || filters.yearTo) {
  where.createdAt = {};
  if (filters.yearFrom) {
    (where.createdAt as any).gte = new Date(`${filters.yearFrom}-01-01`);
  }
  if (filters.yearTo) {
    (where.createdAt as any).lte = new Date(`${filters.yearTo}-12-31T23:59:59`);
  }
}
```

Build dynamic `orderBy` from `sortBy` and `sortOrder`:
- Map `sortBy` to Prisma field: 'title' -> 'title', 'createdAt' -> 'createdAt', 'averageRating' -> 'averageRating'
- Construct `orderBy: { [mappedField]: sortOrder }`

Execute with `Promise.all([findMany, count])` for near-simultaneous pagination:
- findMany: include full relations (use catalogIncludes helper), skip/take for pagination
- count: same where clause

Return `{ entries, total, page: filters.page, limit: filters.limit }`.

**Add to catalog.service.ts:**

`importFromCSV(buffer: Buffer, adminId: string)`:
- Use `parseCSV` helper to parse buffer into rows
- Define expected columns: title, author, publisher, imprint, barcode, isbn, description, seriesTitle (optional — look up series by title)
- Iterate all rows (do NOT stop at first error):
  - Validate each row with `catalogImportRowSchema` from contracts
  - If validation fails, push `{ row: i + 2, message: error messages }` to errors array (i+2 accounts for header + 0-index)
  - If valid, attempt to create catalog entry:
    - If seriesTitle is provided, look up series by title (case-insensitive contains). If not found, push warning but still create entry without series.
    - Create entry with `createdById: adminId`, `approvalStatus: 'DRAFT'`
    - On success, increment `created` counter
    - On error (e.g., duplicate), push to errors array with row number
- Impose a row limit: max 1000 rows per import. If data exceeds 1000, throw BadRequestError("CSV exceeds maximum of 1000 rows")
- Return `{ created: number, errors: Array<{ row: number; message: string }>, total: number }`

`exportToCSV()`:
- Query all APPROVED catalog entries with full includes (series, categories, tags, characters), ordered by title asc
- Map to flat rows: title, author, publisher, imprint, barcode, isbn, description, seriesTitle (from series relation), volumeNumber, editionNumber, categories (joined by '; '), tags (joined by '; '), characters (joined by '; '), averageRating, ratingCount
- Use `generateCSV` helper from shared lib
- Return CSV string

**Add routes in catalog.routes.ts:**

`POST /import` — authenticate, authorize('ADMIN'), uploadCSV('file') middleware, then:
- If no req.file, throw BadRequestError('No CSV file provided')
- Call importFromCSV(req.file.buffer, req.user.id)
- sendSuccess with import results (created count + errors array)

`GET /export` — authenticate, authorize('ADMIN'), then:
- Call exportToCSV()
- Set response headers: `Content-Type: text/csv`, `Content-Disposition: attachment; filename="catalog-export-${Date.now()}.csv"`
- Send CSV string directly: `res.send(csvString)`

IMPORTANT: Both `/import` and `/export` routes must be defined BEFORE `/:id` in the router.

Verify: `pnpm --filter api type-check` and `pnpm --filter api build` must pass.
