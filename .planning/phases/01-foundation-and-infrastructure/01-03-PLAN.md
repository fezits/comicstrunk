<task type="auto">
  <name>Task 1: Configure API and Web for cPanel Passenger deployment with health check and HTTPS</name>
  <files>
    apps/api/src/app.ts
    apps/api/.htaccess
    apps/web/next.config.ts
    scripts/deploy-api.sh
    scripts/deploy-web.sh
    scripts/backup-db.sh
    .env.example
  </files>
  <action>
    **API Passenger compatibility:**
    1. Ensure `apps/api/src/app.ts` creates an http.Server and calls `.listen()` — Passenger captures this call. The pattern from research:
       ```typescript
       const port = process.env.PORT || 3001;
       const server = http.createServer(app);
       server.listen(port, () => { ... });
       ```
    2. Add `app.set('trust proxy', 1)` for correct IP detection behind Apache
    3. Ensure the `/health` endpoint returns: `{ status: 'ok', uptime: process.uptime(), timestamp: new Date().toISOString() }` and also tests the Prisma database connection
    4. Add `/api/v1` prefix to all route registrations

    **HTTPS enforcement via .htaccess:**
    1. Create `apps/api/.htaccess` with Apache rewrite rules:
       ```
       RewriteEngine On
       RewriteCond %{HTTPS} !=on
       RewriteRule ^(.*)$ https://%{HTTP_HOST}/$1 [R=301,L]
       ```
    2. Do NOT add HTTPS redirect in Express middleware — Apache handles SSL termination per research

    **Next.js standalone configuration:**
    1. Update `apps/web/next.config.ts`:
       - `output: 'standalone'`
       - `outputFileTracingRoot: path.join(__dirname, '../../')` (monorepo root)
       - next-intl plugin wrapping (will be fully configured in Plan 06)
    2. Verify standalone build: `pnpm --filter web build` should produce `.next/standalone/`

    **Deployment scripts:**
    1. Create `scripts/deploy-api.sh`:
       - Build: `pnpm --filter api build`
       - Copy dist/ to production path
       - Touch `tmp/restart.txt` to trigger Passenger restart
       - Log deployment timestamp
    2. Create `scripts/deploy-web.sh`:
       - Build: `pnpm --filter web build`
       - Copy `.next/standalone/` to production path
       - Copy `apps/web/public/` into standalone
       - Copy `apps/web/.next/static/` into standalone `.next/static/`
       - Do NOT run pnpm install in standalone (per research anti-pattern)
       - Touch `tmp/restart.txt`

    **Database backup script:**
    1. Create `scripts/backup-db.sh`:
       - Uses mysqldump with credentials from environment variables
       - Saves to a dated backup file
       - Retains last 7 days of backups (deletes older)
       - Meant to be registered as a daily cron job in cPanel

    **Environment variable documentation:**
    1. Create `.env.example` listing all required environment variables with descriptions:
       - DATABASE_URL, JWT_ACCESS_SECRET, JWT_REFRESH_SECRET, WEB_URL, API_URL, NODE_ENV, PORT
       - Clearly marked: "Set these in cPanel UI, NOT in .env files in production"

    **Local verification:**
    - Start the API locally: `pnpm --filter api dev`
    - Verify GET http://localhost:3001/health returns 200 + JSON
    - Build the web app: `pnpm --filter web build`
    - Verify `.next/standalone/server.js` exists
  </action>
  <verify>
    - `pnpm --filter api dev` starts and GET /health returns 200 with uptime and timestamp
    - `pnpm --filter web build` succeeds and `.next/standalone/server.js` exists
    - `cat scripts/deploy-api.sh` contains tmp/restart.txt
    - `cat scripts/deploy-web.sh` contains standalone and does NOT contain "pnpm install"
    - `.env.example` documents all required variables
    - `apps/api/.htaccess` contains HTTPS rewrite rules
  </verify>
  <done>
    API and web are configured for cPanel Passenger deployment. Health endpoint works. HTTPS enforcement is via .htaccess. Deployment and backup scripts exist. Environment variables are documented.
  </done>
