    **Prisma client singleton (`shared/lib/prisma.ts`):**
    - Create singleton pattern that reuses the PrismaClient in dev (avoids too many connections warning)
    - Export `prisma` instance

    **Logger (`shared/lib/logger.ts`):**
    - Create winston logger with JSON format for production, colorized console for development
    - Log levels: error, warn, info, debug
    - Export `logger` instance

    **JWT helpers (`shared/lib/jwt.ts`):**
    - `signAccessToken({ userId, role })`: sign with JWT_ACCESS_SECRET, expiresIn '15m', algorithm HS256
    - `signRefreshToken({ userId, tokenFamily })`: sign with JWT_REFRESH_SECRET, expiresIn '7d'
    - `verifyAccessToken(token)`: verify and return payload or throw
    - `verifyRefreshToken(token)`: verify and return payload or throw
    - SHA-256 hash helper for refresh token storage: `hashToken(token: string): string`
    - Secrets from `process.env.JWT_ACCESS_SECRET` and `process.env.JWT_REFRESH_SECRET` — throw on startup if missing

    **API Error classes (`shared/utils/api-error.ts`):**
    - `ApiError` extends Error with statusCode and isOperational flag
    - Subclasses: `BadRequestError(400)`, `UnauthorizedError(401)`, `ForbiddenError(403)`, `NotFoundError(404)`, `ConflictError(409)`, `TooManyRequestsError(429)`, `InternalError(500)`

    **Response helpers (`shared/utils/response.ts`):**
    - `sendSuccess(res, data, statusCode = 200)`: `{ success: true, data }`
    - `sendError(res, error)`: `{ success: false, error: { message, code } }`
    - `sendPaginated(res, data, pagination)`: `{ success: true, data, pagination: { page, limit, total } }`

    **Authenticate middleware (`shared/middleware/authenticate.ts`):**
    - Extract Bearer token from Authorization header
    - Verify with `verifyAccessToken`
    - Attach `req.user = { userId, role }` to request
    - Return 401 if token missing or invalid

    **Authorize middleware (`shared/middleware/authorize.ts`):**
    - Factory function: `authorize(...roles: UserRole[])` returns middleware
    - Check `req.user.role` is in allowed roles
    - Return 403 if unauthorized

    **Validate middleware (`shared/middleware/validate.ts`):**
    - Factory: `validate(schema: ZodSchema, source: 'body' | 'query' | 'params' = 'body')`
    - Parse request[source] against Zod schema
    - Replace request[source] with parsed (clean) data
    - Return 400 with Zod error details if validation fails

    **Error handler (`shared/middleware/error-handler.ts`):**
    - Catch-all Express error middleware (4 args: err, req, res, next)
    - If ApiError: use its statusCode and message
    - If Prisma known error (P2002 unique constraint): return 409
    - If Zod error: return 400 with formatted issues
    - Otherwise: log and return 500 generic error
    - Never leak stack traces in production

    **Auth service (`modules/auth/auth.service.ts`):**

    `signup(input: SignupInput)`:
    - Validate acceptedTerms is true (AUTH-07)
    - Check email not already taken (prisma.user.findUnique)
    - Hash password with bcryptjs cost 12
    - Create user with role USER and acceptedTermsAt
    - Generate token family (crypto.randomUUID)
    - Sign access token and refresh token
    - Store refresh token hash in DB
    - Return { accessToken, refreshToken, user: { id, name, email, role } }

    `login(input: LoginInput)`:
    - Find user by email
    - Compare password with bcryptjs
    - If invalid: throw UnauthorizedError("Invalid credentials") — same message for both wrong email and wrong password (security)
    - Generate new token family
    - Sign access and refresh tokens
    - Store refresh token hash
    - Return { accessToken, refreshToken, user }

    `refreshTokens(oldRefreshToken: string)`:
    - Verify refresh token JWT
    - Hash the old token and find matching non-revoked record in DB
    - If not found: revoke ALL tokens for that tokenFamily (stolen token detection) and throw UnauthorizedError
    - Revoke old token
    - Sign new access + refresh tokens (SAME tokenFamily)
    - Store new refresh token hash
    - Return { accessToken, refreshToken }

    `logout(refreshToken: string)`:
    - Hash the token, find and revoke it in DB
    - If not found, still return success (idempotent)

    `requestPasswordReset(email: string)`:
    - Find user by email
    - If not found: return success anyway (do not reveal if email exists)
    - Generate crypto-random token (32 bytes hex)
    - Store SHA-256 hash in PasswordReset table with 1h TTL
    - **Phase 1 placeholder:** Log the reset URL to console/logger instead of sending email (full email service comes in Phase 7). Log: `Password reset link: ${WEB_URL}/reset-password?token=${rawToken}`
    - Return success message

    `confirmPasswordReset(token: string, newPassword: string)`:
    - Hash the provided token
    - Find matching PasswordReset where usedAt is null and expiresAt > now
    - If not found: throw BadRequestError("Invalid or expired token")
    - Hash new password, update user's passwordHash
    - Mark PasswordReset as used (set usedAt)
    - Revoke all refresh tokens for this user (force re-login)
    - Return success

    **Auth routes (`modules/auth/auth.routes.ts`):**
    - POST /signup: validate(signupSchema) → signup → set refresh cookie → sendSuccess(201)
    - POST /login: rateLimit(5 per 15min per IP) → validate(loginSchema) → login → set refresh cookie → sendSuccess
    - POST /refresh: read refresh token from cookie → refreshTokens → set new refresh cookie → sendSuccess
    - POST /logout: authenticate → read cookie → logout → clear cookie → sendSuccess
    - POST /password-reset/request: rateLimit(3 per 15min per IP) → validate(resetPasswordRequestSchema) → requestPasswordReset → sendSuccess
    - POST /password-reset/confirm: validate(resetPasswordConfirmSchema) → confirmPasswordReset → sendSuccess

    **Refresh cookie config:**
    ```typescript
    const REFRESH_COOKIE_OPTIONS = {
      httpOnly: true,
      secure: process.env.NODE_ENV === 'production',
      sameSite: 'strict' as const,
      path: '/api/v1/auth/refresh',
      maxAge: 7 * 24 * 60 * 60 * 1000, // 7 days
    };
    ```

    **Rate limiting:**
    - Import express-rate-limit
    - Login limiter: 5 requests per 15 minutes per IP
    - Password reset limiter: 3 requests per 15 minutes per IP
    - Apply per-route, not globally

    **User routes (`modules/users/users.routes.ts`):**
    - GET /profile: authenticate → get user from DB → sendSuccess
    - PUT /profile: authenticate → validate(updateProfileSchema) → update name, bio → sendSuccess
    - The updateProfileSchema allows: name (optional string 2-100), bio (optional string max 500)
    - Avatar upload deferred to later phase per research open question

    **User service (`modules/users/users.service.ts`):**
    - `getProfile(userId)`: prisma.user.findUnique, select safe fields (no passwordHash)
    - `updateProfile(userId, data)`: prisma.user.update with validated fields

    **Wire routes in app.ts:**
    - `app.use('/api/v1/auth', authRoutes)`
    - `app.use('/api/v1/users', usersRoutes)`
    - Error handler must be registered LAST

    **Import contracts schemas:**
    - Import signupSchema, loginSchema, resetPasswordRequestSchema, resetPasswordConfirmSchema from @comicstrunk/contracts
    - Use in validate() middleware calls
