Purpose: Automates the subscription lifecycle so plan changes propagate instantly from Stripe events without manual intervention. The webhook is the source of truth for subscription state changes.
Output: Stripe webhook endpoint at /api/v1/webhooks/stripe with idempotent event processing for 5 event types.
</objective>

  <name>Task 1: Stripe webhook service with idempotent event handlers</name>
  <files>

  <action>
Create `apps/api/src/modules/subscriptions/stripe-webhook.service.ts` implementing the full webhook processing pipeline.

**Main function `processStripeWebhook(rawBody: Buffer, signature: string)`**:
1. Call `verifyWebhookSignature(rawBody, signature)` from shared/lib/stripe.ts to get the Stripe event
2. **Idempotency guard**: Insert into `prisma.webhookEvent.create({ provider: 'stripe', eventId: event.id, eventType: event.type, payload: event.data as Prisma.InputJsonValue })`. Catch Prisma P2002 (unique violation) and return silently (duplicate event).
3. Route event to handler based on `event.type`:
   - `checkout.session.completed` -> handleCheckoutCompleted
   - `customer.subscription.updated` -> handleSubscriptionUpdated
   - `customer.subscription.deleted` -> handleSubscriptionDeleted
   - `invoice.payment_failed` -> handlePaymentFailed
   - `invoice.paid` -> handleInvoicePaid
4. Mark as processed: `prisma.webhookEvent.update({ where: { provider_eventId: { provider: 'stripe', eventId: event.id } }, data: { processedAt: new Date() } })`

**`handleCheckoutCompleted(session: Stripe.Checkout.Session)`**:
- Extract userId from `session.metadata?.comicstrunkUserId`. If missing or mode !== 'subscription', return.
- Retrieve full subscription from Stripe: `stripe.subscriptions.retrieve(session.subscription as string)`
- Use `prisma.$transaction` to:
  a. Upsert Subscription: findFirst by userId where status IN ('ACTIVE', 'TRIALING', 'PAST_DUE') with stripeSubscriptionId matching, or create new. Set: stripeCustomerId=session.customer, stripeSubscriptionId=stripeSubscription.id, planType='BASIC', status=(stripeSubscription.status==='trialing' ? 'TRIALING' : 'ACTIVE'), currentPeriodStart/End from stripeSubscription timestamps (multiply by 1000 for Date)
  b. Update user role to SUBSCRIBER (unless already ADMIN)
- Log: `[Stripe Webhook] Checkout completed for user ${userId}`

**`handleSubscriptionUpdated(stripeSubscription: Stripe.Subscription)`**:
- Find local subscription by stripeSubscriptionId
- If not found, log warning and return (out-of-order event)
- Map Stripe status to local enum: 'active'->'ACTIVE', 'trialing'->'TRIALING', 'past_due'->'PAST_DUE', 'canceled'/'cancelled'->'CANCELLED'
- Update: status, currentPeriodStart, currentPeriodEnd
- If stripeSubscription.cancel_at_period_end === true, set cancelledAt to new Date() (marks end-of-period cancellation)
- If stripeSubscription.cancel_at_period_end === false and local cancelledAt was set, clear cancelledAt to null (user reactivated)

**`handleSubscriptionDeleted(stripeSubscription: Stripe.Subscription)`**:
- Find local subscription by stripeSubscriptionId. If not found, return.
- Use `prisma.$transaction` to:
  a. Update subscription: status='CANCELLED', planType='FREE', cancelledAt=new Date()
  b. Find user. If user.role !== 'ADMIN', update role to 'USER'
- Create notification: type='SUBSCRIPTION_EXPIRED', title='Assinatura expirada', message='Sua assinatura BASIC expirou. Seu plano foi revertido para FREE.'

**`handlePaymentFailed(invoice: Stripe.Invoice)`**:
- Find local subscription by stripeSubscriptionId (invoice.subscription as string). If not found, return.
- Update subscription status to 'PAST_DUE'
- Create notification: type='SUBSCRIPTION_PAYMENT_FAILED', title='Falha no pagamento da assinatura', message='Houve uma falha no pagamento da sua assinatura BASIC. Por favor, atualize seu meio de pagamento para evitar a perda dos beneficios.'

**`handleInvoicePaid(invoice: Stripe.Invoice)`**:
- Find local subscription by stripeSubscriptionId. If not found, return.
- Only process if subscription status is 'PAST_DUE' or 'TRIALING' (renewal recovery or trial-to-paid)
- Retrieve full subscription from Stripe to get fresh period dates
- Update local subscription: status='ACTIVE', currentPeriodStart/End from Stripe data
- If user role was reverted to USER, set back to SUBSCRIBER

All handlers must be wrapped in try/catch with console.error logging so one handler failure doesn't crash the webhook endpoint.
