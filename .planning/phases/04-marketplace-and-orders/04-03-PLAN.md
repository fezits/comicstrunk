<task type="auto">
  <name>Task 1: Create shipping service for addresses, methods, and tracking</name>
  <files>
    apps/api/src/modules/shipping/shipping.service.ts
  </files>
  <action>
    Create the shipping service covering SHIP-01 through SHIP-05:

    **Address CRUD (SHIP-01, SHIP-02):**

    `createAddress(userId, data)`:
    - If data.isDefault is true, use $transaction to: (1) unset all existing defaults for this user, (2) create the new address with isDefault=true
    - If no addresses exist for user, auto-set isDefault=true regardless of input
    - Return the created address

    `listAddresses(userId)`:
    - Find all addresses for user, ordered by isDefault desc, createdAt desc
    - Return array

    `getAddress(userId, addressId)`:
    - Find by id; throw NotFoundError if not found
    - Verify address.userId === userId; throw ForbiddenError if mismatch
    - Return address

    `updateAddress(userId, addressId, data)`:
    - Find and verify ownership
    - If data.isDefault is true, use $transaction to unset others then update
    - Return updated address

    `deleteAddress(userId, addressId)`:
    - Find and verify ownership
    - If this was the default, after deletion, set the most recent remaining address as default (if any)
    - Delete and return success

    `setDefaultAddress(userId, addressId)`:
    - Use $transaction: unset all defaults for userId, then set the specified one
    - Return updated address

    **Shipping Methods (SHIP-03):**

    `listShippingMethods()`: Return all shipping methods, ordered by name
    `listActiveShippingMethods()`: Return only isActive=true methods
    `createShippingMethod(data)`: Admin creates a method (name, description, isActive)
    `updateShippingMethod(id, data)`: Admin updates a method
    `deleteShippingMethod(id)`: Admin deletes a method

    **Tracking Update (SHIP-04, SHIP-05):**

    `updateTracking(sellerId, orderItemId, data: { trackingCode, carrier })`:
    - Find OrderItem by id, include order
    - Verify orderItem.sellerId === sellerId; throw ForbiddenError if not the seller
    - Verify orderItem.status is 'PROCESSING' (only valid state for adding tracking); throw BadRequestError if wrong status
    - Update trackingCode, carrier, shippedAt = now(), status = 'SHIPPED'
    - Return updated order item
    - SHIP-05 (buyer notified of shipping updates) defers entirely to Phase 7 (Community and Notifications). For now, just log the event: console.log(`[SHIPPING] Tracking updated for order item ${orderItemId}`) as a hook point for future notification integration.
  </action>

  </verify>

  </done>

    **apps/api/src/modules/shipping/shipping.routes.ts:**

    Address routes (all authenticated):
    - `POST /addresses` — Validate body with createShippingAddressSchema. Call createAddress. Return 201.
    - `GET /addresses` — List user's addresses. Call listAddresses.
    - `GET /addresses/:id` — Get single address. Call getAddress.
    - `PUT /addresses/:id` — Validate body with updateShippingAddressSchema. Call updateAddress.
    - `DELETE /addresses/:id` — Call deleteAddress.
    - `PATCH /addresses/:id/default` — Call setDefaultAddress. (No body needed.)

    Shipping method routes:
    - `GET /methods` — Public. List active shipping methods (call listActiveShippingMethods).
    - `GET /methods/all` — Admin only. List all methods including inactive.
    - `POST /methods` — Admin only. Validate with createShippingMethodSchema. Call createShippingMethod. Return 201.
    - `PUT /methods/:id` — Admin only. Validate with updateShippingMethodSchema (partial). Call updateShippingMethod.
    - `DELETE /methods/:id` — Admin only. Call deleteShippingMethod.

    Tracking route:
    - `PATCH /tracking/:orderItemId` — Authenticated (seller). Validate body with updateTrackingSchema. Call updateTracking(req.user.userId, req.params.orderItemId, body).

    Place static routes (/methods, /methods/all) before parameterized routes.
    Export router with `Router` type annotation.

    **apps/api/src/app.ts:**
    - Import shippingRouter from modules/shipping/shipping.routes
    - Mount: `app.use('/api/v1/shipping', shippingRouter)`
