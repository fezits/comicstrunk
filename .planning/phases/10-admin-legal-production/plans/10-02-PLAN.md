# Plan 10-02: LGPD Compliance API

## Wave: 1 (parallel with 10-01)

## Goal
LGPD data rights: access/export user data as JSON, request corrections, schedule account deletion with fiscal data preservation.

## Tasks

### 1. Create contracts (`packages/contracts/src/lgpd.ts`)
- createDataRequestSchema — type (ACCESS | CORRECTION | DELETION | EXPORT), details (optional string)
- DataRequestType, DataRequestStatus enums
- Export types + add to index.ts

### 2. Create LGPD service (`apps/api/src/modules/lgpd/lgpd.service.ts`)
- `createDataRequest(userId, data)` — create DataRequest, prevent duplicate pending requests of same type
- `listUserRequests(userId)` — user's own requests
- `listAllRequests(filters)` — admin: all requests paginated
- `processRequest(requestId, adminId)` — admin marks as PROCESSING
- `completeRequest(requestId, adminId)` — admin marks as COMPLETED
- `rejectRequest(requestId, adminId, reason)` — admin rejects with reason

- `exportUserData(userId)` — assemble ALL user data as JSON:
  - Profile (name, email, bio, social links)
  - Collection items (title, condition, price, notes)
  - Orders (as buyer: order number, items, total, dates)
  - Reviews (rating, text, date)
  - Comments (text, date)
  - Favorites (catalog entry titles)
  - Notifications preferences
  - Shipping addresses
  - Subscription history
  - Legal acceptances
  - Do NOT include: passwordHash, refreshTokens, internal IDs

- `scheduleAccountDeletion(userId)` — create DataRequest type=DELETION, status=PENDING
  - Send confirmation email
  - Actual deletion happens via cron after 30 days

- `executeAccountDeletion(userId)` — called by cron:
  - Delete: refreshTokens, passwordResets, cartItems, shippingAddresses, bankAccounts, notifications, notificationPreferences, favorites, commentLikes, clickLogs
  - Anonymize: reviews (set userId to 'deleted'), comments (set userId to 'deleted')
  - Preserve: orders, disputes, subscriptions, legalAcceptances, dataRequests
  - Anonymize user: set name='Conta Excluída', email=uuid@deleted.comicstrunk.com, passwordHash='', clear all optional fields
  - Set role to USER (prevent admin deletion)

### 3. Add deletion cron
- In cron/index.ts: daily at 4 AM, find PENDING DELETION requests older than 30 days, execute deletion

### 4. Create LGPD routes (`apps/api/src/modules/lgpd/lgpd.routes.ts`)
Authenticated:
- `POST /requests` — createDataRequest
- `GET /requests` — listUserRequests
- `GET /export` — exportUserData (returns JSON download)
- `POST /delete-account` — scheduleAccountDeletion

Admin:
- `GET /admin/requests` — listAllRequests
- `PUT /admin/requests/:id/process` — processRequest
- `PUT /admin/requests/:id/complete` — completeRequest
- `PUT /admin/requests/:id/reject` — rejectRequest

### 5. Register routes in create-app.ts
- Mount at `/api/v1/lgpd`

## Verification
- User can request data export (JSON download)
- User can schedule account deletion
- Admin can manage data requests
- Deletion cron anonymizes correctly (preserves fiscal data)
