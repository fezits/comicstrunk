  <files>

  <action>
1. **Create `apps/api/src/modules/comments/comments.service.ts`**:

   **`createComment(userId, data: CreateCommentInput)`**
   - Verify catalogEntryId exists and is APPROVED
   - If parentId is provided:
     - Fetch parent comment, verify it exists
     - Verify parent comment belongs to the same catalogEntryId
     - **Enforce one nesting level**: If parent.parentId is not null, throw BadRequestError('Replies can only be one level deep')
   - Create comment with userId, catalogEntryId, parentId (or null), content
   - Return created comment with user info

   **`updateComment(userId, commentId, data: UpdateCommentInput)`**
   - Find comment by id, verify it exists
   - Verify comment.userId === userId (only author can edit)
   - Update content field
   - Return updated comment

   **`deleteComment(userId, commentId)`**
   - Find comment by id, verify it exists
   - Verify comment.userId === userId (only author can delete)
   - Delete comment (cascade will remove likes and replies via Prisma onDelete)

   **`getCatalogComments(catalogEntryId, page, limit, currentUserId?: string)`**
   - Fetch top-level comments (parentId IS NULL) where catalogEntryId matches
   - Include: user (name, avatarUrl), replies (include user name/avatar, likes count), likesCount
   - If currentUserId provided, also check if user has liked each comment (for UI toggle state)
   - Order by createdAt desc
   - Return { comments, total, page, limit }

   **`toggleCommentLike(userId, commentId)`**
   - Find existing like: CommentLike where userId + commentId
   - If exists: delete it, decrement comment.likesCount
   - If not exists: create it, increment comment.likesCount
   - Use a transaction to ensure likesCount stays consistent
   - Return { liked: boolean, likesCount: number }

2. **Create `apps/api/src/modules/comments/comments.routes.ts`**:

   Routes:
   - `POST /` (authenticate, validate(createCommentSchema)) ‚Äî creates comment or reply, returns 201
   - `PUT /:id` (authenticate, validate(updateCommentSchema)) ‚Äî updates comment content
   - `DELETE /:id` (authenticate) ‚Äî deletes comment
   - `GET /catalog/:catalogEntryId` (public, validate query with catalogCommentsQuerySchema) ‚Äî paginated comments with nested replies. If user is authenticated (optional authenticate), include like status.
   - `POST /:id/like` (authenticate) ‚Äî toggles like on comment

   Export as `commentsRoutes: Router`
  </action>
