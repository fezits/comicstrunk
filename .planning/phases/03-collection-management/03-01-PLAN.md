3. **Collection service** (`apps/api/src/modules/collection/collection.service.ts`):

   a. **Refactor `checkPlanLimit`** to return structured data instead of just throwing:
      - Change the function to return `{ currentCount, limit, planType }` for use in error messages and in the transaction.
      - When throwing `BadRequestError`, include a second argument with `{ currentCount: Number(count), limit, planType }` so the error handler can forward it. Example: `throw new BadRequestError('Collection limit reached (...)', { currentCount, limit, planType })`.

   b. **Refactor `addItem`** to use `prisma.$transaction` for atomicity (COLL-09):
      - Wrap the entire addItem logic (catalog entry check, duplicate check, plan limit check, and create) inside `prisma.$transaction(async (tx) => { ... })`.
      - Inside the transaction, use `tx.collectionItem.count()`, `tx.catalogEntry.findUnique()`, `tx.collectionItem.findFirst()`, and `tx.collectionItem.create()` instead of `prisma.*`.
      - The plan limit check inside the transaction should use `tx.subscription.findFirst()` to get the user's plan.
      - This ensures the count and insert are serialized within the same transaction, preventing the race condition.

   c. **Refactor `importCSV`** similarly:
      - Wrap the entire import loop inside `prisma.$transaction(async (tx) => { ... })`.
      - Use `tx.*` for all queries inside the loop.
      - Check plan limit at the start of the transaction using `tx.collectionItem.count()`.
      - This prevents concurrent imports from exceeding the limit.

   d. **Add `addPhoto` function**:
      - Signature: `addPhoto(userId: string, itemId: string, photoUrl: string): Promise<CollectionItem>`
      - Verify item exists and belongs to user.
      - Read current `photoUrls` (parse JSON array or default to `[]`). Max 5 photos per item.
      - Append the new URL and update the item: `prisma.collectionItem.update({ where: { id: itemId }, data: { photoUrls: [...currentPhotos, photoUrl] } })`.
      - Return updated item with includes.

   e. **Add `removePhoto` function**:
      - Signature: `removePhoto(userId: string, itemId: string, photoIndex: number): Promise<CollectionItem>`
      - Verify item exists and belongs to user.
      - Parse `photoUrls`, remove at index, update.
      - Import and call `deleteImage` from cloudinary to clean up the removed photo (extract publicId from URL).
      - Return updated item with includes.

   f. **Add `getMissingEditions` function**:
      - Signature: `getMissingEditions(userId: string, seriesId: string)`
      - Find all APPROVED catalog entries in the series: `prisma.catalogEntry.findMany({ where: { seriesId, approvalStatus: 'APPROVED' }, select: { id, title, editionNumber, volumeNumber, coverImageUrl, slug }, orderBy: { editionNumber: 'asc' } })`.
      - Find user's collection items for this series: `prisma.collectionItem.findMany({ where: { userId, catalogEntry: { seriesId } }, select: { catalogEntryId: true } })`.
      - Build a `Set` of owned catalogEntryIds.
      - Return `allEditions.filter(e => !ownedSet.has(e.id))`.
